# Statistical Arbitrage

Statistical arbitrage (stat arb) extends pairs trading from two assets to many. Instead of trading single pairs, we construct portfolios of related assets, betting that mispricings across the portfolio will correct.

The key insight: more assets means more diversification. A single pair can diverge permanently; a portfolio of 50 pairs is far less likely to all diverge at once.

---

## 8.7 Portfolio-Based Stat Arb

### 8.7.1 Prose/Intuition

Portfolio-based stat arb uses statistical techniques to identify mean-reverting combinations of assets:

**Principal Component Analysis (PCA):** Decompose returns into orthogonal factors. The first few components capture market/sector movements; the later components may capture mean-reverting residuals.

**Eigenportfolio trading:** Trade portfolios constructed from eigenvectors. Lower eigenvalue portfolios often mean-revert because they represent idiosyncratic noise rather than persistent trends.

**Basket trading:** Instead of trading pairs, trade one asset against a basket of similar assets. If Coca-Cola diverges from the "beverage sector" as a whole, it's more likely a temporary mispricing than a fundamental change.

The advantage: diversification reduces the impact of any single relationship breaking down.

### 8.7.3 Mathematical Derivation

**PCA Decomposition:**

Given returns matrix $R$ (T × N), compute the covariance matrix:

$$
\Sigma = \frac{1}{T-1} R^T R
$$

Eigendecomposition:
$$
\Sigma = V \Lambda V^T
$$

where $V$ contains eigenvectors (principal components) and $\Lambda$ is diagonal with eigenvalues.

**Eigenportfolio Construction:**

The $k$-th eigenportfolio has weights $w_k = V_k / \|V_k\|_1$ (normalised eigenvector).

Portfolio returns:
$$
r_k = w_k^T r_t
$$

**Mean Reversion of Later Components:**

If returns follow a factor model:
$$
r_i = \beta_i F + \varepsilon_i
$$

The first PC captures $F$ (market factor). Later PCs capture $\varepsilon$ (idiosyncratic), which is more likely to mean-revert.

**Statistical Test:**

Test if eigenportfolio returns are stationary using variance ratio or ADF tests. Lower eigenvalue portfolios tend to have lower variance ratios (more mean-reverting).

## 8.8 ETF Arbitrage

### 8.8.1 Prose/Intuition

ETFs should trade at their Net Asset Value (NAV)—the value of their underlying holdings. When they don't, arbitrage opportunities arise:

**Premium:** ETF price > NAV
- Arbitrageurs sell (short) the ETF, buy the underlying basket
- Forces ETF price down toward NAV

**Discount:** ETF price < NAV
- Arbitrageurs buy the ETF, sell (short) the underlying basket
- Forces ETF price up toward NAV

The creation/redemption mechanism allows authorised participants to exchange ETF shares for underlying baskets, ensuring prices stay aligned.

For retail traders, small premiums/discounts may exist intraday. These are typically too small to profit from after costs, but illustrate the mean-reversion principle.

### 8.8.2 Visual Evidence


``` r
# Approximate SPY premium/discount using component stocks
```

### 8.8.3 Mathematical Derivation

**Net Asset Value:**

$$
\text{NAV} = \frac{\sum_{i} n_i \times P_i}{\text{Shares Outstanding}}
$$

where $n_i$ is the number of shares of stock $i$ in the ETF and $P_i$ is its price.

**Premium/Discount:**

$$
\text{Premium} = \frac{P_{\text{ETF}} - \text{NAV}}{\text{NAV}}
$$

**Arbitrage Bounds:**

Transaction costs create a no-arbitrage band:

$$
\text{NAV} - c_{\text{creation}} < P_{\text{ETF}} < \text{NAV} + c_{\text{redemption}}
$$

where $c$ includes bid-ask spreads, commissions, and creation/redemption fees.

**Creation/Redemption:**

Authorised participants can:
- **Create:** Exchange basket of stocks for ETF shares (when ETF trades at premium)
- **Redeem:** Exchange ETF shares for basket of stocks (when ETF trades at discount)

This mechanism anchors ETF prices to NAV.

### 8.8.4 Implementation & Application


``` r
# Simple ETF arbitrage strategy
# Calculate returns first (before filtering)
spy_nav[, spy_ret := c(NA, diff(log(SPY)))]
spy_nav[, nav_ret := c(NA, diff(log(nav_approx)))]
spy_nav[, spread_ret := spy_ret - nav_ret]  # Return on SPY - NAV position

# Filter after calculating returns
spy_nav_strat <- spy_nav[!is.na(spread_ret) & !is.na(premium)]

# Strategy: when premium > threshold, short SPY/long basket (expect convergence)
threshold <- 0.5  # 0.5% premium/discount

# Initialize positions
n_rows <- nrow(spy_nav_strat)
positions <- numeric(n_rows)
position <- 0

if (n_rows > 1) {
    for (i in 2:n_rows) {
        prem <- spy_nav_strat$premium[i - 1]

        if (position == 0) {
            if (prem > threshold) position <- -1     # Short spread (expect premium to fall)
            else if (prem < -threshold) position <- 1  # Long spread (expect discount to close)
        } else if (position == -1 && prem < 0) {
            position <- 0
        } else if (position == 1 && prem > 0) {
            position <- 0
        }

        positions[i] <- position
    }
}

spy_nav_strat[, strat_ret := -position * spread_ret]  # Negative because short spread means profit when spread falls

# Cumulative
spy_nav_strat[, cum_strat := exp(cumsum(strat_ret)) - 1]
```

## 8.9 Risk Management for Stat Arb

### 8.9.1 Prose/Intuition

Statistical arbitrage is *not* risk-free arbitrage. Relationships can break permanently:

**Structural breaks:**
- Mergers, acquisitions, spin-offs change company fundamentals
- Regulatory changes affect industries differently
- Technology disruption (e.g., Netflix vs Blockbuster)

**Regime changes:**
- Correlations spike during crises (everything falls together)
- Relationships that worked in calm markets fail in stress

**Model decay:**
- Parameters estimated from history may not hold in the future
- Overfitting to historical patterns

**Crowding:**
- Many funds trade similar strategies
- Crowded trades can unwind violently when positions need to be closed

Risk management is essential: position limits, stop-losses, and monitoring for relationship breakdown.

``` r
# Monitor half-life over time using XOM/CVX energy pair
xom <- load_market("XOM")
cvx <- load_market("CVX")

xom <- filter_dates(xom, as.Date("2015-01-01"), as.Date("2023-12-31"))
cvx <- filter_dates(cvx, as.Date("2015-01-01"), as.Date("2023-12-31"))

pair <- merge(xom[, .(date, XOM = log(adjusted))], cvx[, .(date, CVX = log(adjusted))], by = "date")

# Rolling OU parameter estimation
window <- 252
pair[, spread := XOM - mean(XOM/CVX) * CVX]  # Simplified spread

rolling_halflife <- numeric(nrow(pair))
rolling_halflife[1:(window-1)] <- NA

for (i in window:nrow(pair)) {
    spread_window <- pair$spread[(i - window + 1):i]

    # OU regression
    delta <- diff(spread_window)
    lag <- spread_window[-length(spread_window)]

    if (length(unique(lag)) > 1) {
        fit <- lm(delta ~ lag)
        theta <- -coef(fit)["lag"]
        if (!is.na(theta) && theta > 0) {
            rolling_halflife[i] <- log(2) / theta
        } else {
            rolling_halflife[i] <- NA  # Not mean-reverting
        }
    }
}

pair[, halflife := rolling_halflife]
pair <- pair[!is.na(halflife)]

# Cap at reasonable values for display
pair[, halflife_capped := pmin(halflife, 100)]

ggplot(pair, aes(x = date, y = halflife_capped)) +
    geom_line(colour = tc[1], linewidth = 0.6) +
    geom_hline(yintercept = c(10, 30), linetype = "dashed", colour = c(tc[3], tc[4])) +
    labs(title = "Rolling Half-Life (XOM/CVX)",
         subtitle = "Green: fast reversion; Red: slow/no reversion",
```

### 8.9.3 Mathematical Derivation

**Stop-Loss Rules:**

For spread $z$:
$$
\text{Stop} = |z_{\text{stop}}| \times \hat{\sigma}
$$

Common choice: stop at $z = 4$ (2× entry threshold).

**Half-Life Monitoring:**

Track $t_{1/2} = \ln(2)/\theta$ over rolling windows. Alert when:
$$
t_{1/2}^{\text{current}} > 2 \times t_{1/2}^{\text{historical}}
$$

**Rolling Cointegration:**

Re-test cointegration on rolling windows. If ADF statistic rises above critical value:
- Relationship may have broken
- Consider reducing position or exiting

**Position Limits:**

Risk per pair:
$$
\text{Position}_{\text{max}} = \frac{\text{Capital} \times \text{Max Risk \%}}{\text{Stop Distance} \times \text{Volatility}}
$$

### 8.9.4 Implementation & Application


``` r
# Risk-managed pairs trading using XOM/CVX
risk_pair <- merge(
    xom[, .(date, XOM = log(adjusted))],
    cvx[, .(date, CVX = log(adjusted))],
    by = "date"
)

# Calculate spread with rolling parameters
window <- 60
risk_pair[, spread := XOM - frollmean(XOM/CVX, window) * CVX]
risk_pair[, spread_mean := frollmean(spread, window)]
risk_pair[, spread_std := frollapply(spread, window, sd)]
risk_pair[, z := (spread - spread_mean) / spread_std]

# Returns
risk_pair[, xom_ret := c(NA, diff(XOM))]
risk_pair[, cvx_ret := c(NA, diff(CVX))]
risk_pair[, spread_ret := xom_ret - mean(risk_pair$XOM/risk_pair$CVX, na.rm = TRUE) * cvx_ret]

risk_pair <- risk_pair[!is.na(z) & !is.na(spread_ret)]

# Strategy with stop-loss
entry_threshold <- 2
stop_threshold <- 4

position <- 0
positions <- numeric(nrow(risk_pair))

for (i in 2:nrow(risk_pair)) {
    z_prev <- risk_pair$z[i - 1]

    # Skip NA values
    if (is.na(z_prev)) {
        positions[i] <- position
        next
    }

    # Check for stop-loss
    if (abs(z_prev) > stop_threshold && position != 0) {
        position <- 0  # Stop out
    }
    # Entry/exit logic
    else if (position == 0) {
        if (z_prev < -entry_threshold) position <- 1
        else if (z_prev > entry_threshold) position <- -1
    }
    else if (position == 1 && z_prev > 0) position <- 0
    else if (position == -1 && z_prev < 0) position <- 0

    positions[i] <- position
}

risk_pair[, position := positions]
risk_pair[, strat_ret := position * spread_ret]

# Also run without stop-loss for comparison
position_no_stop <- 0
positions_no_stop <- numeric(nrow(risk_pair))

for (i in 2:nrow(risk_pair)) {
    z_prev <- risk_pair$z[i - 1]

    # Skip NA values
    if (is.na(z_prev)) {
        positions_no_stop[i] <- position_no_stop
        next
    }

    if (position_no_stop == 0) {
        if (z_prev < -entry_threshold) position_no_stop <- 1
        else if (z_prev > entry_threshold) position_no_stop <- -1
    }
    else if (position_no_stop == 1 && z_prev > 0) position_no_stop <- 0
    else if (position_no_stop == -1 && z_prev < 0) position_no_stop <- 0

    positions_no_stop[i] <- position_no_stop
}

risk_pair[, position_no_stop := positions_no_stop]
risk_pair[, strat_ret_no_stop := position_no_stop * spread_ret]

# Cumulative returns
risk_pair[, cum_stop := exp(cumsum(strat_ret)) - 1]
risk_pair[, cum_no_stop := exp(cumsum(strat_ret_no_stop)) - 1]

# Statistics
cat("=== Risk-Managed Pairs Trading ===\n")
```

## Quick Reference

### PCA-Based Stat Arb

| Component | Formula | Notes |
|-----------|---------|-------|
| Covariance | $\Sigma = R^T R / (T-1)$ | From return matrix |
| Eigenportfolio | $w_k = V_k / \|V_k\|_1$ | Normalised eigenvector |
| Variance explained | $\lambda_k / \sum \lambda$ | Importance of PC |

### ETF Arbitrage

| Metric | Formula | Action |
|--------|---------|--------|
| Premium | $(P_{ETF} - NAV)/NAV$ | Short ETF, long basket |
| Discount | $(NAV - P_{ETF})/NAV$ | Long ETF, short basket |
| Arbitrage bound | $|P - NAV| > c$ | Trade when outside |

### Risk Management

| Risk | Monitor | Action |
|------|---------|--------|
| Spread divergence | $|z| > 4$ | Stop-loss |
| Relationship decay | Half-life increase | Reduce position |
| Cointegration break | Rolling ADF test | Exit |
| Crowding | Volume/correlation | Diversify |

### Key Guidelines

1. **Diversify pairs:** Single pair = idiosyncratic risk; many pairs = systematic edge
2. **Monitor relationships:** Test cointegration regularly; parameters drift
3. **Use stop-losses:** Spreads can diverge permanently; limit losses
4. **Mind the costs:** Stat arb is capital-intensive; transaction costs matter
5. **Beware crowding:** Popular strategies attract capital; returns compress
