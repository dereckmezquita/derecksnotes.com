## 1. Prose: From Signals to Portfolios

> "A great signal poorly integrated into a portfolio is worth less than a mediocre signal well integrated. Portfolio construction is where alpha meets reality."

We've covered individual components: statistical foundations, factor models, volatility trading, machine learning, execution quality, and risk management. Now we synthesize everything into a complete system. This integration is where most quantitative efforts fail—not from bad models but from poor assembly.

### The Integration Challenge

Consider what a portfolio manager must reconcile:

**Multiple Signals**: A momentum signal says buy, a value signal says sell, and a volatility signal suggests reduce exposure. How do you combine them?

**Competing Objectives**: Maximum Sharpe ratio? Minimum drawdown? Tracking error constraint? Turnover limits? These often conflict.

**Implementation Reality**: Your optimal portfolio requires 50% turnover, but transaction costs make that prohibitive. The paper portfolio diverges from the live portfolio.

**Regime Dependency**: The signal combination that works in trending markets fails in mean-reverting regimes. Static weights become liability.

### The Complete Pipeline

A production-grade system follows this flow:

```
Data → Features → Signals → Scores → Weights → Orders → Execution → P&L → Risk
  ↑                                                                          |
  └──────────────────────────────── Feedback Loop ───────────────────────────┘
```

Each stage has its own challenges:
- **Data**: Quality, timing, survivorship bias
- **Features**: Normalization, decay, non-stationarity
- **Signals**: Combination, correlation, capacity
- **Scores**: Ranking, z-scoring, winsorization
- **Weights**: Optimization, constraints, robustness
- **Orders**: Sizing, urgency, venue selection
- **Execution**: Market impact, timing, slippage
- **Risk**: Limits, attribution, monitoring

Today we build the complete integration layer.

``` r
equal_combine <- function(signals) {
    rowMeans(signals, na.rm = TRUE)

ic_weighted_combine <- function(signals, ics) {
    # Normalize ICs to weights
    ics_positive <- pmax(ics, 0)  # Only use positive IC signals
    if (sum(ics_positive) == 0) ics_positive <- rep(1, length(ics))
```

## 3. Mathematical Derivation: Optimal Portfolio Construction

### Mean-Variance Optimization with Constraints

The classic Markowitz problem:

$$\max_w \; w^T \mu - \frac{\lambda}{2} w^T \Sigma w$$

Subject to:
- $\sum_i w_i = 0$ (dollar neutral)
- $|w_i| \leq w_{\max}$ (position limits)
- $\sum_i |w_i| \leq L$ (leverage constraint)
- $|w^T \beta| \leq \beta_{\max}$ (factor exposure limits)

The unconstrained solution:

$$w^* = \frac{1}{\lambda} \Sigma^{-1} \mu$$

With constraints, we use quadratic programming or penalized optimization.

### Alpha Combination

Given $K$ alpha signals with forecasts $\alpha_k$ and information coefficients $IC_k$:

**Equal Weighted**:
$$\alpha_{combined} = \frac{1}{K} \sum_{k=1}^K z(\alpha_k)$$

**IC Weighted**:
$$\alpha_{combined} = \sum_{k=1}^K \frac{IC_k}{\sum_j IC_j} z(\alpha_k)$$

**Optimal (Grinold)**:
$$\alpha_{combined} = \sigma_{\text{target}} \cdot IC_{combined} \cdot z(\alpha)$$

Where $IC_{combined} = \sqrt{\sum_k IC_k^2}$ if signals are uncorrelated.

### Transaction Cost Integration

With linear transaction costs $c$ per unit traded, the optimization becomes:

$$\max_w \; w^T \mu - \frac{\lambda}{2} w^T \Sigma w - c \cdot \|w - w_{\text{prev}}\|_1$$

The $L_1$ penalty creates a "no-trade region" around current positions, trading off alpha capture against transaction costs.

**Optimal Trade-Off**:

Only trade when: $\text{Expected Alpha Gain} > 2 \times \text{Transaction Cost}$

$$|\alpha_{new} - \alpha_{old}| > 2c$$

### Risk Parity Alternative

Instead of maximizing Sharpe, equalize risk contribution:

$$RC_i = w_i \cdot \frac{(\Sigma w)_i}{\sqrt{w^T \Sigma w}}$$

Set $RC_i = \frac{1}{N}$ total risk for all $i$. This leads to:

$$w_i \propto \frac{1}{\sigma_i \cdot \sqrt{\sum_j \rho_{ij} \cdot w_j / w_i}}$$

For uncorrelated assets: $w_i \propto 1/\sigma_i$ (inverse volatility).

## 4. Implementation: Production Portfolio System


``` r
#' Complete portfolio optimizer with constraints
#' @param alpha vector of alpha forecasts
#' @param covariance covariance matrix
#' @param prev_weights previous portfolio weights
#' @param constraints list of constraints
#' @return optimized portfolio weights
optimize_portfolio <- function(
    alpha,
    covariance,
    prev_weights = NULL,
    constraints = list(
        max_position = 0.10,
        max_leverage = 2.0,
        dollar_neutral = TRUE,
        turnover_penalty = 0.001
    )
) {
    n <- length(alpha)
    if (is.null(prev_weights)) prev_weights <- rep(0, n)

    # Simple approach: alpha-weighted with constraints
    # (Full QP optimization would require quadprog package)

    # Step 1: Z-score alpha
    alpha_z <- (alpha - mean(alpha)) / sd(alpha)

    # Step 2: Initial weights proportional to alpha
    raw_weights <- alpha_z / sum(abs(alpha_z))

    # Step 3: Apply position limits
    constrained <- pmin(pmax(raw_weights, -constraints$max_position),
                        constraints$max_position)

    # Step 4: Dollar neutral adjustment
    if (constraints$dollar_neutral) {
        long_sum <- sum(constrained[constrained > 0])
        short_sum <- sum(constrained[constrained < 0])

        # Scale to equal long/short exposure
        target_exposure <- min(abs(long_sum), abs(short_sum))
        if (long_sum > 0) {
            constrained[constrained > 0] <- constrained[constrained > 0] *
                                             (target_exposure / long_sum)
        }
        if (short_sum < 0) {
            constrained[constrained < 0] <- constrained[constrained < 0] *
                                             (target_exposure / abs(short_sum))
        }
    }

    # Step 5: Leverage constraint
    current_leverage <- sum(abs(constrained))
    if (current_leverage > constraints$max_leverage) {
        constrained <- constrained * (constraints$max_leverage / current_leverage)
    }

    # Step 6: Turnover consideration (shrink toward previous weights)
    if (constraints$turnover_penalty > 0) {
        blend <- 0.9  # How much to blend with prev weights
        constrained <- blend * constrained + (1 - blend) * prev_weights
    }

    # Normalize final weights
    final_weights <- constrained / sum(abs(constrained)) * constraints$max_leverage

    # Ensure dollar neutral
    if (constraints$dollar_neutral) {
        final_weights <- final_weights - mean(final_weights)
    }

    data.table(
        asset = 1:n,
        alpha = alpha,
        alpha_z = alpha_z,
        raw_weight = raw_weights,
        final_weight = final_weights,
        position_type = ifelse(final_weights > 0.001, "Long",
                              ifelse(final_weights < -0.001, "Short", "Flat"))
    )
}
```

``` r
#' Complete backtest engine with realistic assumptions
#' @param returns matrix of asset returns (days x assets)
#' @param signals matrix of signals (days x assets)
#' @param params backtest parameters
#' @return data.table of backtest results
backtest_portfolio <- function(
    returns,
    signals,
    params = list(
        rebalance_freq = 5,  # Days between rebalances
        max_position = 0.10,
        transaction_cost = 0.001,  # 10 bps per trade
        slippage = 0.0005  # 5 bps slippage
    )
) {
    n_days <- nrow(returns)
    n_assets <- ncol(returns)

    # Initialize
    weights <- matrix(0, n_days, n_assets)
    portfolio_value <- numeric(n_days)
    portfolio_value[1] <- 1

    daily_returns <- numeric(n_days)
    turnover <- numeric(n_days)
    costs <- numeric(n_days)

    current_weights <- rep(0, n_assets)

    for (t in 2:n_days) {
        # Check if rebalance day
        if ((t - 1) %% params$rebalance_freq == 0 && t > 50) {
            # Get signal for this day
            signal_t <- signals[t, ]

            if (all(!is.na(signal_t))) {
                # Optimize new weights
                opt <- optimize_portfolio(
                    alpha = signal_t,
                    covariance = NULL,
                    prev_weights = current_weights,
                    constraints = list(
                        max_position = params$max_position,
                        max_leverage = 2.0,
                        dollar_neutral = TRUE,
                        turnover_penalty = 0.001
                    )
                )
                target_weights <- opt$final_weight
            } else {
                target_weights <- current_weights
            }
        } else {
            target_weights <- current_weights
        }

        # Calculate turnover
        trade_size <- abs(target_weights - current_weights)
        turnover[t] <- sum(trade_size) / 2

        # Calculate transaction costs
        total_cost <- sum(trade_size) * (params$transaction_cost + params$slippage)
        costs[t] <- total_cost

        # Update weights
        current_weights <- target_weights
        weights[t, ] <- current_weights

        # Calculate returns
        asset_returns <- returns[t, ]
        if (any(is.na(asset_returns))) asset_returns[is.na(asset_returns)] <- 0

        gross_return <- sum(current_weights * asset_returns)
        daily_returns[t] <- gross_return - total_cost

        # Update portfolio value
        portfolio_value[t] <- portfolio_value[t-1] * (1 + daily_returns[t])
    }

    # Compile results
    data.table(
        day = 1:n_days,
        portfolio_value = portfolio_value,
        daily_return = daily_returns,
        turnover = turnover,
        costs = costs,
        gross_exposure = rowSums(abs(weights)),
        net_exposure = rowSums(weights),
        n_positions = rowSums(abs(weights) > 0.001)
    )
}
```

``` r
# Monte Carlo simulation of strategy outcomes
n_simulations <- 500
n_forward_days <- 252

# Estimate strategy parameters from backtest
mu <- mean(bt_results$daily_return)
sigma <- sd(bt_results$daily_return)

# Run simulations
simulation_paths <- matrix(NA, n_forward_days, n_simulations)

for (sim in 1:n_simulations) {
    sim_returns <- rnorm(n_forward_days, mean = mu, sd = sigma)
    simulation_paths[, sim] <- cumprod(1 + sim_returns)
}

# Calculate percentiles
percentiles <- t(apply(simulation_paths, 1, quantile,
                       probs = c(0.05, 0.25, 0.50, 0.75, 0.95)))
colnames(percentiles) <- c("p05", "p25", "p50", "p75", "p95")

sim_summary <- data.table(
    day = 1:n_forward_days,
    as.data.table(percentiles)
)

ggplot(sim_summary, aes(x = day)) +
    geom_ribbon(aes(ymin = p05, ymax = p95), fill = tc["neutral"], alpha = 0.2) +
    geom_ribbon(aes(ymin = p25, ymax = p75), fill = tc["neutral"], alpha = 0.3) +
    geom_line(aes(y = p50), color = tc["positive"], linewidth = 1) +
    geom_hline(yintercept = 1, linetype = "dashed", color = tc["negative"]) +
    labs(
        title = "Monte Carlo Forward Simulation (1 Year)",
        subtitle = "Median path with 50% and 90% confidence intervals",
        x = "Forward Day",
        y = "Portfolio Value"
    ) +
```

## Summary

Portfolio integration is where statistical research becomes investable reality:

**Signal Combination**:
- Multiple weak signals combine into stronger composite
- IC-weighted combination adapts to regime shifts
- Correlation between signals affects combination benefit

**Portfolio Construction**:
- Mean-variance with constraints is the foundation
- Transaction costs create no-trade regions
- Risk parity offers robust alternative to MVO

**Implementation Bridge**:
- Paper portfolios must account for execution reality
- Turnover drives costs; optimize for net returns
- Monte Carlo reveals range of outcomes, not just expected

**Complete Pipeline**:
```
Data → Features → Signals → Combination → Optimization →
Orders → Execution → P&L → Risk Monitoring → Feedback
```

This completes our journey from statistical foundations through portfolio integration. The tools and frameworks presented provide the building blocks for systematic, quantitative investment management. The edge comes not from any single technique but from rigorous integration across the entire pipeline.

Remember: in finance, the competition is fierce, the data is noisy, and yesterday's alpha is tomorrow's crowded trade. Continuous research, robust risk management, and disciplined execution separate sustainable strategies from fleeting anomalies.
