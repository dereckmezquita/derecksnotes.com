## 12.1 The Execution Gap

### 12.1.1 Prose & Intuition

The gap between backtest returns and live trading returns is called **slippage**. This gap destroys more strategies than any other factor. A backtest assumes perfect execution—buying at exactly the price you want, when you want, in the size you want. Reality is messier.

**Market impact** is the price movement caused by your own trading. Large orders move prices against you. The more you trade, the worse your fills become.

**Timing delay** between signal generation and order execution costs money. Markets move while you're deciding what to do.

**Partial fills** occur when liquidity isn't sufficient for your order size. You might only get half your desired position at the intended price.

Understanding and modelling these costs is essential for any systematic strategy. A strategy that looks profitable ignoring costs may be deeply unprofitable once execution is realistic.

### 12.1.3 Mathematical Derivation

**Net Return After Costs**

$$
r_{net} = r_{gross} - c \cdot \text{turnover}
$$

where $c$ is cost per unit traded and turnover is $|w_t - w_{t-1}|$.

**Impact on Sharpe Ratio**

$$
SR_{net} = \frac{\mu - c \cdot \bar{T}}{\sigma}
$$

where $\bar{T}$ is average turnover. Higher turnover requires higher gross returns to maintain positive Sharpe.

**Break-Even Cost**

Strategy breaks even when:

$$
c_{BE} = \frac{\mu}{\bar{T}}
$$

For a strategy with 10% annual return and 10x turnover:

$$
c_{BE} = \frac{0.10}{10} = 1\% = 100 \text{ bps}
$$

**Cost Drag on Returns**

Annual cost drag:

$$
\text{Cost Drag} = c \cdot T_{annual}
$$

For 20 bps costs and 50x annual turnover:

$$
\text{Cost Drag} = 0.002 \times 50 = 10\%
$$

``` r
# Cost = spread + market impact

estimate_cost <- function(trade_size_pct, spread_bps = 1, volatility, avg_volume) {
    # Spread cost (always pay at least half the spread)
    spread_cost <- spread_bps / 2

    # Market impact: square root model
    # Impact = k * sigma * sqrt(trade_size / avg_volume)
    k <- 0.1  # Impact coefficient
    impact <- k * volatility * sqrt(trade_size_pct / 100) * 10000

    return(spread_cost + impact)
}
```

## 12.2 Market Impact Models

### 12.2.1 Prose & Intuition

Market impact is the price movement caused by your trading. When you buy, you push the price up; when you sell, you push it down. This is unavoidable for any strategy that trades in size.

**Temporary impact** is the immediate price move that reverts after your trading. It's caused by consuming liquidity from the order book.

**Permanent impact** is the price move that persists after trading. It represents information incorporated into the market price.

The standard model assumes impact scales with the **square root** of trade size. This reflects the microstructure of limit order books: small trades consume only the best prices; large trades must reach deeper into the book.

Understanding impact is crucial for:
1. Estimating realistic backtest costs
2. Optimising execution schedules
3. Sizing positions appropriately

``` r
# Almgren-Chriss optimal execution trajectory
# Trade X shares over time T

almgren_chriss <- function(X, T, n_periods, sigma, eta, gamma, lambda) {
    # Parameters:
    # X = total shares to trade
    # T = trading horizon
    # sigma = volatility
    # eta = temporary impact
    # gamma = permanent impact
    # lambda = risk aversion

    dt <- T / n_periods
    kappa <- sqrt(lambda * sigma^2 / eta)

    # Optimal trajectory
    t <- seq(0, T, length.out = n_periods + 1)
    x_t <- X * sinh(kappa * (T - t)) / sinh(kappa * T)

    return(data.table(time = t / T, remaining = x_t / X))
}

# Different risk aversion levels
trajectories <- rbind(
    cbind(almgren_chriss(1000, 1, 100, 0.02, 0.001, 0.0001, 0.001), lambda = "Low (Patient)"),
    cbind(almgren_chriss(1000, 1, 100, 0.02, 0.001, 0.0001, 0.01), lambda = "Medium"),
    cbind(almgren_chriss(1000, 1, 100, 0.02, 0.001, 0.0001, 0.1), lambda = "High (Urgent)")
)

ggplot(trajectories, aes(x = time, y = remaining, colour = lambda)) +
    geom_line(linewidth = 1) +
    scale_colour_manual(values = c(tc[1], tc[2], tc[4])) +
    scale_y_continuous(labels = scales::percent_format()) +

         subtitle = "Higher risk aversion = faster execution; lower impact = slower",
```

### 12.2.3 Mathematical Derivation

**Square Root Market Impact**

$$
\text{Impact} = \sigma \cdot \sqrt{\frac{V_{trade}}{V_{daily}}}
$$

where $\sigma$ is daily volatility, $V_{trade}$ is trade volume, $V_{daily}$ is average daily volume.

**Almgren-Chriss Model**

Cost function to minimise:

$$
C(x) = \gamma \sum_k n_k + \eta \sum_k n_k^2 + \lambda \sum_k \sigma^2 x_k^2 \Delta t
$$

where:
- $\gamma$ = permanent impact coefficient
- $\eta$ = temporary impact coefficient
- $\lambda$ = risk aversion parameter
- $n_k$ = shares traded in period $k$
- $x_k$ = remaining position

**Optimal Execution Trajectory**

$$
x_t = X \cdot \frac{\sinh(\kappa(T-t))}{\sinh(\kappa T)}
$$

where $\kappa = \sqrt{\frac{\lambda \sigma^2}{\eta}}$

**VWAP Tracking**

$$
\text{VWAP Error} = \frac{\sum_k p_k \cdot v_k}{\sum_k v_k} - \text{Benchmark VWAP}
$$

``` r
# Simulate execution with realistic market dynamics
simulate_execution <- function(total_shares, strategy, n_periods = 10,
                               vol = 0.02, impact_k = 0.1) {

    # Initial price
    price <- 100

    # Volume profile (U-shaped)
    volume_profile <- (1:n_periods - n_periods/2)^2 + 5
    volume_profile <- volume_profile / sum(volume_profile)

    # Execute according to strategy
    if (strategy == "TWAP") {
        schedule <- rep(1/n_periods, n_periods)
    } else if (strategy == "VWAP") {
        schedule <- volume_profile
    } else if (strategy == "Front-loaded") {
        schedule <- rev(1:n_periods)
        schedule <- schedule / sum(schedule)
    } else {
        schedule <- rep(1/n_periods, n_periods)
    }

    shares_per_period <- total_shares * schedule

    # Simulate execution
    results <- data.table(
        period = 1:n_periods,
        shares = shares_per_period
    )

    results[, impact := impact_k * sqrt(shares / (total_shares / n_periods)) * vol * 100]
    results[, price_move := rnorm(.N, 0, vol / sqrt(n_periods)) * 100]
    results[, exec_price := 100 + cumsum(price_move) + cumsum(impact)]

    results[, cost := shares * impact / total_shares]

    return(list(
        schedule = results,
        total_cost = sum(results$cost),
        avg_price = sum(results$exec_price * results$shares) / total_shares
    ))
}
```

## 12.3 Transaction Cost Analysis

### 12.3.1 Prose & Intuition

Transaction Cost Analysis (TCA) decomposes execution costs into their components. This helps identify where costs are coming from and how to reduce them.

**Implementation Shortfall** measures the difference between the decision price (when you decided to trade) and the actual execution price. It captures all costs: spread, impact, and timing.

Breaking down implementation shortfall:
1. **Delay cost**: Price move between decision and order
2. **Trading cost**: Spread and impact during execution
3. **Opportunity cost**: Cost of shares not executed

Good TCA helps answer:
- Are our execution algorithms performing well?
- Which brokers provide best execution?
- Should we trade faster or slower?

### 12.3.2 Visual Evidence


``` r
# Simulate and decompose implementation shortfall
set.seed(42)

# Simulate a buy order
decision_price <- 100
order_size <- 1000
n_fills <- 10

# Simulate price movement and fills
fills <- data.table(
    fill_num = 1:n_fills,
    shares = rep(order_size / n_fills, n_fills),
    delay = cumsum(runif(n_fills, 0.01, 0.05)),  # Time in days
    price_drift = cumsum(rnorm(n_fills, 0.01, 0.1)),  # Price drift
    spread = runif(n_fills, 0.01, 0.03),  # Spread
    impact = sqrt(1:n_fills / n_fills) * 0.05  # Increasing impact
)

fills[, fill_price := decision_price + price_drift + spread + impact]

# Decomposition
avg_fill <- sum(fills$fill_price * fills$shares) / order_size
delay_cost <- fills[1, price_drift]  # Drift before first fill

# Create breakdown
breakdown <- data.table(
    Component = c("Decision Price", "Delay Cost", "Spread", "Impact", "Avg Fill Price"),
    Value = c(decision_price, delay_cost, mean(fills$spread), mean(fills$impact), avg_fill)

# Waterfall chart
```

### 12.3.3 Mathematical Derivation

**Implementation Shortfall**

$$
IS = \frac{P_{avg} - P_{decision}}{P_{decision}} \times \text{Side}
$$

where Side = 1 for buys, -1 for sells.

**Decomposition**

$$
IS = \text{Delay} + \text{Trading} + \text{Opportunity}
$$

**Delay Cost**

$$
\text{Delay} = \frac{P_{arrival} - P_{decision}}{P_{decision}} \times \text{Side}
$$

**Trading Cost**

$$
\text{Trading} = \frac{P_{avg} - P_{arrival}}{P_{decision}} \times \frac{Q_{filled}}{Q_{total}} \times \text{Side}
$$

**Opportunity Cost**

$$
\text{Opportunity} = \frac{P_{close} - P_{decision}}{P_{decision}} \times \frac{Q_{unfilled}}{Q_{total}} \times \text{Side}
$$
