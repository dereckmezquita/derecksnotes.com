# Chapter 1 -- Describing Data

## Part 1: Types of Data and Central Tendency

Statistics begins with data. Before we can build models, test hypotheses, or make predictions, we must first understand *what kind* of data we have and how to summarise it. This chapter develops the foundational vocabulary and tools for describing datasets -- the building blocks upon which every subsequent chapter rests.

We work with two real datasets throughout this chapter: the **Palmer Penguins** dataset (measurements of three penguin species in Antarctica) and the **NHANES** dataset (a nationally representative health survey from the United States). Both are messy, real-world datasets with missing values, skewed distributions, and interesting structure -- exactly the sort of data a practising statistician encounters daily.

## 1.1 Types of Data

Every variable in a dataset belongs to one of two broad families: **categorical** or **numerical**. Getting this classification right matters enormously, because the type of variable determines which summary statistics are meaningful, which visualisations are appropriate, and which statistical tests are valid.

### 1.1.1 Categorical Variables

A **categorical variable** (also called a *qualitative* variable) records a label or category rather than a quantity. There are two sub-types:

- **Nominal variables** have categories with no natural ordering. Examples: species of penguin, blood type, country of residence. It makes no sense to say that "Adelie" is greater than "Chinstrap."

- **Ordinal variables** have categories with a meaningful order, but the distances between categories are not necessarily equal. Examples: education level (primary < secondary < tertiary), pain severity (mild < moderate < severe), Likert-scale responses (strongly disagree < disagree < neutral < agree < strongly agree).

Let us examine some categorical variables in the penguins dataset:

### 1.1.2 Numerical Variables

A **numerical variable** (also called a *quantitative* variable) records a measurement or count. Again, there are two sub-types:

- **Discrete variables** can take only specific, separated values (usually integers). Examples: number of eggs in a nest, number of hospital visits, count of bacterial colonies on a plate. You can have 3 eggs or 4 eggs, but not 3.7 eggs.

- **Continuous variables** can, in principle, take any value within an interval. Examples: body mass in grams, flipper length in millimetres, blood pressure in mmHg. A penguin's flipper might be 181.0 mm, 181.3 mm, or 181.27 mm -- the precision is limited only by the measuring instrument.

### 1.1.3 Why the Distinction Matters

The type of variable constrains what operations are valid:

| Operation | Nominal | Ordinal | Discrete | Continuous |
|-----------|---------|---------|----------|------------|
| Count frequencies | Yes | Yes | Yes | Yes |
| Compute mean | No | Debatable | Yes | Yes |
| Compute median | No | Yes | Yes | Yes |
| Compute variance | No | No | Yes | Yes |
| Rank-order | No | Yes | Yes | Yes |
| Arithmetic (+, -, etc.) | No | No | Yes | Yes |

Computing the "mean" of a set of species names is nonsensical. Computing the mean of an ordinal variable (e.g., assigning 1 = mild, 2 = moderate, 3 = severe) is common in practice but controversial -- it assumes the gaps between categories are equal, which may not be true.

## 1.2 Measures of Central Tendency

Given a collection of numerical values, the most basic question is: *where is the centre?* A **measure of central tendency** provides a single number that attempts to represent the "typical" value. We shall study three: the mean, the median, and the mode.

### 1.2.1 The Arithmetic Mean

The **arithmetic mean** (or simply "the mean") is the sum of all values divided by the number of values. It is the most commonly used measure of centre and has deep connections to probability theory that we shall explore in later chapters.

**Notation.** Let $x_1, x_2, \ldots, x_n$ denote $n$ observations. The sample mean is:

$$\bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i = \frac{x_1 + x_2 + \cdots + x_n}{n}$$

In plain English: add up all the values, then divide by how many there are. The symbol $\bar{x}$ (read "x-bar") is the standard notation for a sample mean. When we refer to the mean of the entire population, we use the Greek letter $\mu$ (mu).

**Physical interpretation.** The mean is the *balance point* of the data. If you placed each observation as an equal weight on a number line, the mean is the point at which the line would balance perfectly. Equivalently, the sum of deviations from the mean is always zero:

$$\sum_{i=1}^{n}(x_i - \bar{x}) = 0$$

This is not merely a nice property -- it is the *defining* property. The mean is the unique value $c$ that minimises the sum of squared deviations $\sum(x_i - c)^2$.

#### Building `my_mean()` from scratch

Let us implement the mean ourselves, first using an explicit loop and then using vectorised operations:


``` r
# Version 1: explicit loop
my_mean_loop <- function(x) {
  x <- x[!is.na(x)]  # remove missing values
  n <- length(x)
  if (n == 0) return(NA_real_)

  total <- 0
  for (i in seq_len(n)) {
    total <- total + x[i]
  }
  total / n
}

# Version 2: vectorised (idiomatic R)
my_mean <- function(x) {
  x <- x[!is.na(x)]
  n <- length(x)
  if (n == 0) return(NA_real_)
  sum(x) / n
}

# Test on penguin body mass
my_mean_loop(penguins$body_mass_g)
my_mean(penguins$body_mass_g)
mean(penguins$body_mass_g, na.rm = TRUE)
```

```
[1] 4201.754
[1] 4201.754
[1] 4201.754
```

All three return the same result. The vectorised version is preferred in R because it is both more concise and faster (R's `sum()` is implemented in C).

#### When the mean is misleading

The mean is sensitive to extreme values. A single very large or very small observation can pull the mean substantially away from the "typical" value. Consider income data: in a room of 50 people with average income \$50,000, adding one billionaire raises the mean to approximately \$20 million -- hardly representative of anyone in the room.
