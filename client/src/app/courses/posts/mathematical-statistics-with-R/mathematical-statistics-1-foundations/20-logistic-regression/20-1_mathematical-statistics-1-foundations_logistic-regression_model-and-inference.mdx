# Chapter 20 -- Logistic Regression

## Part 1: Model and Inference

Logistic regression is the most widely used model for binary outcomes. In Chapter 19 we developed the GLM framework and saw that the Bernoulli distribution belongs to the exponential family with canonical link equal to the logit. In this chapter we take that abstract machinery and make it concrete: we derive the logistic model from first principles, interpret its parameters in terms of odds and odds ratios, fit it with `glm()`, and conduct formal inference using Wald and likelihood ratio tests.

---

## 20.1 From Exponential Family to Logistic Regression

### 20.1.1 The Derivation

Recall from Chapter 19 that the Bernoulli PMF in exponential family form is:

$$P(Y = y \mid p) = \exp\!\left[y \log\frac{p}{1-p} + \log(1-p)\right]$$

The canonical parameter is $\theta = \log[p/(1-p)]$, the **logit** of $p$. The GLM links this to the linear predictor:

$$\log\frac{p_i}{1 - p_i} = \eta_i = \beta_0 + \beta_1 x_{i1} + \cdots + \beta_p x_{ip}$$

Solving for $p_i$:

$$p_i = \frac{\exp(\eta_i)}{1 + \exp(\eta_i)} = \frac{1}{1 + \exp(-\eta_i)}$$

This is the **logistic function** (or **sigmoid**), and the resulting model is **logistic regression**.

### 20.1.2 Why the Logit Link Is Natural

The logit link arises from requiring $p \in (0, 1)$ while allowing the linear predictor $\eta$ to range over $(-\infty, \infty)$. The logistic function maps the real line onto the unit interval in a smooth, monotone, symmetric fashion.

## 20.2 Odds and Log-Odds

### 20.2.1 Definitions

The **odds** of an event with probability $p$ are:

$$\text{Odds} = \frac{p}{1 - p}$$

If $p = 0.75$, the odds are $3:1$ -- the event is three times as likely to occur as not. The **log-odds** (logit) are simply the natural logarithm of the odds:

$$\text{logit}(p) = \log\frac{p}{1-p}$$

The logistic regression model says that the log-odds are a linear function of the predictors.

### 20.2.2 Odds Ratios

The **odds ratio** (OR) for a one-unit increase in $x_j$ is:

$$\text{OR}_j = \frac{\text{Odds}(x_j + 1)}{\text{Odds}(x_j)} = e^{\beta_j}$$

This is the key interpretive tool for logistic regression: $e^{\beta_j}$ tells us how the odds multiply for every one-unit increase in $x_j$, holding all other predictors constant.

- $\text{OR} > 1$: the odds increase (risk factor).
- $\text{OR} = 1$: no effect.
- $\text{OR} < 1$: the odds decrease (protective factor).

## 20.5 Maximum Likelihood Estimation

### 20.5.1 The Log-Likelihood

For independent binary observations, the log-likelihood is:

$$\ell(\boldsymbol{\beta}) = \sum_{i=1}^n \bigl[y_i \log p_i + (1 - y_i) \log(1 - p_i)\bigr]$$

where $p_i = \text{logit}^{-1}(\mathbf{x}_i^\top \boldsymbol{\beta})$. This is the negative of the **binary cross-entropy** loss used in machine learning.

### 20.5.2 Computing the Log-Likelihood from Scratch


``` r
loglik_logistic <- function(beta, X, y) {
  eta <- X %*% beta
  p <- 1 / (1 + exp(-eta))
  p <- pmax(pmin(p, 1 - 1e-15), 1e-15)
  sum(y * log(p) + (1 - y) * log(1 - p))
}

X <- model.matrix(~ lwt + age + smoke, data = bwt)
y <- bwt$low

c(
  "Log-lik (scratch)" = loglik_logistic(coef(fit1), X, y),
  "Log-lik (R)" = as.numeric(logLik(fit1))
)
```

## 20.6 Wald Tests

The **Wald test** for $H_0: \beta_j = 0$ uses the fact that, asymptotically:

$$z_j = \frac{\hat{\beta}_j}{\text{SE}(\hat{\beta}_j)} \sim N(0, 1)$$

R's `summary.glm()` reports these $z$-values and their two-sided $p$-values.

### 20.6.1 Wald Confidence Intervals

The $(1 - \alpha)$ Wald CI for $\beta_j$ is:

$$\hat{\beta}_j \pm z_{\alpha/2} \cdot \text{SE}(\hat{\beta}_j)$$

And the CI for the odds ratio is obtained by exponentiating:

$$\exp\!\bigl[\hat{\beta}_j \pm z_{\alpha/2} \cdot \text{SE}(\hat{\beta}_j)\bigr]$$



## 20.7 Likelihood Ratio Tests

The **likelihood ratio test** (LRT) is generally preferred over Wald tests because it is more reliable in small samples and is invariant to reparametrisation.

$$\Lambda = -2[\ell(\hat{\boldsymbol{\beta}}_{\text{reduced}}) - \ell(\hat{\boldsymbol{\beta}}_{\text{full}})] \sim \chi^2_{q}$$

where $q$ is the number of parameters dropped.


``` r
fit_full <- glm(low ~ lwt + age + smoke + ht + ui, data = bwt, family = binomial)
fit_no_smoke <- glm(low ~ lwt + age + ht + ui, data = bwt, family = binomial)

lr_stat <- deviance(fit_no_smoke) - deviance(fit_full)
p_lr <- pchisq(lr_stat, df = 1, lower.tail = FALSE)

data.frame(
  Test = "H0: beta_smoke = 0",
  LR_statistic = round(lr_stat, 4),
  df = 1,
  p_value = round(p_lr, 4)
)
```

## 20.10 Goodness of Fit: Hosmer-Lemeshow Test

The Hosmer-Lemeshow test groups observations by predicted probability and compares observed and expected counts in each group using a chi-squared statistic.


``` r
hosmer_lemeshow <- function(y, p_hat, g = 10) {
  breaks <- quantile(p_hat, probs = seq(0, 1, length.out = g + 1))
  breaks[1] <- breaks[1] - 0.001
  group <- cut(p_hat, breaks = breaks, labels = FALSE)

  obs1 <- tapply(y, group, sum)
  n_g <- tapply(y, group, length)
  exp1 <- tapply(p_hat, group, sum)

  hl_stat <- sum((obs1 - exp1)^2 / (exp1 * (1 - exp1 / n_g)))
  p_val <- pchisq(hl_stat, df = g - 2, lower.tail = FALSE)
  list(statistic = hl_stat, df = g - 2, p_value = p_val)
}

hl <- hosmer_lemeshow(bwt$low, fitted(fit_multi))
data.frame(
  Statistic = round(hl$statistic, 3),
  df = hl$df,
  p_value = round(hl$p_value, 3)
)
```

A large $p$-value suggests no evidence of poor fit (the model adequately describes the data).

## Communicating to Stakeholders

When presenting logistic regression results:

- **Use odds ratios, not coefficients.** Say "smoking multiplies the odds of low birth weight by 2.4" rather than "the log-odds coefficient for smoking is 0.88."
- **Always provide confidence intervals.** An OR of 2.4 with a CI of (0.9, 6.1) tells a very different story from an OR of 2.4 with a CI of (1.8, 3.2).
- **Show predicted probability curves.** Audiences understand probabilities far more naturally than log-odds.
- **Clarify "holding other variables constant."** This is the most common source of misunderstanding. Emphasise that the OR for smoking compares smokers and non-smokers *who are otherwise identical*.

| Concept | Details |
|:--------|:--------|
| **Model** | $\text{logit}(p_i) = \mathbf{x}_i^\top \boldsymbol{\beta}$ |
| **Logistic function** | $p = 1 / (1 + e^{-\eta})$ |
| **Odds ratio** | $\text{OR}_j = e^{\beta_j}$ |
| **Log-likelihood** | $\sum [y_i \log p_i + (1-y_i) \log(1-p_i)]$ |
| **Wald test** | $z = \hat{\beta}_j / \text{SE}(\hat{\beta}_j)$ |
| **LRT** | $D_{\text{reduced}} - D_{\text{full}} \sim \chi^2$ |
| **Profile CI** | Invert the LRT (preferred over Wald) |
| **R syntax** | `glm(y ~ x, family = binomial)` |

## Exercises

1. **Derivation.** Starting from $P(Y=1) = e^{\eta}/(1+e^{\eta})$, show that $\partial p / \partial \eta = p(1-p)$. This is the key identity for deriving the score equations.

2. **Interpretation.** In a logistic regression for disease status, the coefficient for age is $\hat{\beta} = 0.03$. What is the odds ratio for a 10-year increase in age? Compute the Wald 95% CI for this odds ratio if $\text{SE}(\hat{\beta}) = 0.01$.

3. **Centering.** Fit `glm(low ~ lwt + age + smoke, family = binomial)` and then fit the same model with `lwt` and `age` centred at their means. Compare the intercepts and verify that the slope coefficients are identical.

4. **Wald vs LRT.** For the multivariate birthwt model, compare the Wald $p$-value for each predictor with the LRT $p$-value obtained by dropping that predictor. When do they disagree?

5. **Separation.** Create a dataset with 20 observations where the two classes are perfectly separated by $x$. Fit logistic regression and observe what happens to the coefficients and standard errors. Why?

6. **Profile likelihood.** For the smoking coefficient in the birthwt model, plot the profile log-likelihood manually (as we did in Section 20.5.3) and mark the profile 95% CI endpoints. Verify they match `confint()`.
