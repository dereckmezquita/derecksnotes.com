# Chapter 9 -- Interval Estimation

## Part 1: Confidence Intervals for Means

In Chapter 8 we learned to produce a single number -- a point estimate -- for an unknown parameter. A point estimate is a best guess, but it tells us nothing about how *precise* that guess is. If someone tells you "the average blood pressure is 120 mmHg," your first question should be: "plus or minus how much?"

An **interval estimate** answers that question. Instead of a single number, we produce a *range* of plausible values together with a statement about how confident we are that the range contains the true parameter. This is the **confidence interval** (CI), and it is arguably the most useful -- and most misunderstood -- concept in all of applied statistics.

---

## 9.1 What a Confidence Interval IS and IS NOT

### 9.1.1 The Correct Interpretation

Suppose we construct a 95% CI for a population mean $\mu$ and obtain the interval $(4.2, 7.8)$. What does this mean?

**Correct interpretation:** If we were to repeat the entire experiment -- draw a new sample, compute a new CI -- over and over again, then 95% of the intervals so constructed would contain the true value of $\mu$.

The confidence level refers to the *procedure*, not to the particular interval in hand. Our specific interval $(4.2, 7.8)$ either contains $\mu$ or it does not -- there is no randomness left once the data are observed. The 95% is a statement about the long-run reliability of the *method*.

### 9.1.2 Common Misinterpretations

The following statements are **wrong**, yet they appear routinely in published papers, textbooks, and news articles:

1. **"There is a 95% probability that $\mu$ lies in $(4.2, 7.8)$."** This is wrong because $\mu$ is a fixed (though unknown) constant. It does not have a probability distribution. The interval is random before the data are collected; $\mu$ is not.

2. **"95% of the data fall in this interval."** A CI is about the *parameter*, not about individual data points. A prediction interval addresses data coverage; a CI does not.

3. **"If we repeat the experiment, 95% of our new observations will fall in this interval."** Same error as above -- confusing a CI with a prediction interval.

4. **"We are 95% sure the interval is correct."** This anthropomorphises the procedure. The interval is deterministic once computed; "sureness" does not apply.

### 9.1.3 Why the Distinction Matters

The difference between "95% of intervals capture $\mu$" and "there is a 95% chance $\mu$ is in this interval" may seem pedantic, but it has real consequences. In clinical trials, regulators base decisions on whether a CI for a treatment effect excludes zero. If the CI is misinterpreted, the decision is misinterpreted. In Bayesian statistics (Chapter 22) we will meet the **credible interval**, which *does* allow a direct probability statement -- but it requires a prior distribution. The frequentist CI makes no such assumption, and in exchange it requires the more careful interpretation above.

## 9.2 The Z-Interval (Known $\sigma$)

### 9.2.1 Setup and Assumptions

Suppose $X_1, X_2, \ldots, X_n \overset{\text{iid}}{\sim} N(\mu, \sigma^2)$, where $\mu$ is unknown but $\sigma$ is **known**. This is rarely realistic -- we seldom know $\sigma$ without knowing $\mu$ -- but it provides the cleanest derivation and sets the pattern for everything that follows.

### 9.2.2 Standardisation

From Chapter 7 (the Central Limit Theorem), we know that:

$$Z = \frac{\bar{X} - \mu}{\sigma / \sqrt{n}} \sim N(0, 1).$$

This is exact when the data are Normal and approximately true for large $n$ regardless of the parent distribution.

### 9.2.3 Derivation

We want to find an interval $(L, U)$ such that $P(\mu \in (L, U)) = 1 - \alpha$, where $\alpha$ is a small number (typically 0.05 for a 95% CI).

Start from the probability statement about $Z$:

$$P\!\left(-z_{\alpha/2} \le \frac{\bar{X} - \mu}{\sigma/\sqrt{n}} \le z_{\alpha/2}\right) = 1 - \alpha,$$

where $z_{\alpha/2}$ is the upper $\alpha/2$ quantile of the standard Normal (i.e., $P(Z > z_{\alpha/2}) = \alpha/2$).

Multiply all three parts of the inequality by $\sigma/\sqrt{n}$:

$$P\!\left(-z_{\alpha/2} \frac{\sigma}{\sqrt{n}} \le \bar{X} - \mu \le z_{\alpha/2} \frac{\sigma}{\sqrt{n}}\right) = 1 - \alpha.$$

Subtract $\bar{X}$ from all three parts and multiply by $-1$ (which reverses the inequalities):

$$P\!\left(\bar{X} - z_{\alpha/2} \frac{\sigma}{\sqrt{n}} \le \mu \le \bar{X} + z_{\alpha/2} \frac{\sigma}{\sqrt{n}}\right) = 1 - \alpha.$$

This gives us the **Z-interval**:

$$\bar{X} \pm z_{\alpha/2} \frac{\sigma}{\sqrt{n}}.$$

The quantity $z_{\alpha/2} \cdot \sigma/\sqrt{n}$ is called the **margin of error**.

### 9.2.4 Implementation from Scratch


``` r
set.seed(42)

#' Z-interval for a population mean (known sigma)
#' @param x numeric vector of observations
#' @param sigma known population standard deviation
#' @param conf confidence level (default 0.95)
#' @return named vector: lower, upper, mean, margin
my_z_interval <- function(x, sigma, conf = 0.95) {
  n <- length(x)
  xbar <- mean(x)
  alpha <- 1 - conf
  z <- qnorm(1 - alpha / 2)
  me <- z * sigma / sqrt(n)
  c(lower = xbar - me, upper = xbar + me, mean = xbar, margin = me)
}

# Generate data from N(50, 10^2) -- pretend sigma = 10 is known
x <- rnorm(30, mean = 50, sd = 10)
my_z_interval(x, sigma = 10)
```

```
    lower     upper      mean    margin 
47.107480 54.264257 50.685868  3.578388 
```

The sample mean is close to 50, and the interval is centred on it with a margin of about $\pm 3.6$.

## 9.3 The t-Interval (Unknown $\sigma$)

### 9.3.1 Why We Need It

In practice, $\sigma$ is almost never known. When we replace $\sigma$ with the sample standard deviation $S$, the resulting statistic is no longer standard Normal:

$$T = \frac{\bar{X} - \mu}{S / \sqrt{n}} \sim t_{n-1}.$$

This is Student's t-distribution with $n - 1$ degrees of freedom, which we derived in Chapter 5 (Section 5.6). The t-distribution has heavier tails than the Normal -- reflecting the additional uncertainty introduced by estimating $\sigma$.

### 9.3.2 Derivation

The argument follows the Z-interval derivation with $t_{n-1,\,\alpha/2}$ replacing $z_{\alpha/2}$ and $S$ replacing $\sigma$:

$$P\!\left(\bar{X} - t_{n-1,\,\alpha/2} \frac{S}{\sqrt{n}} \le \mu \le \bar{X} + t_{n-1,\,\alpha/2} \frac{S}{\sqrt{n}}\right) = 1 - \alpha.$$

The **t-interval** is therefore:

$$\bar{X} \pm t_{n-1,\,\alpha/2} \frac{S}{\sqrt{n}}.$$

For large $n$ the t-distribution converges to the Normal, so the Z-interval and t-interval give nearly identical results. For small $n$ the t-interval is wider -- appropriately so, because our estimate of spread is less precise with fewer observations.

### 9.3.3 Implementation from Scratch


``` r
#' t-interval for a population mean (unknown sigma)
#' @param x numeric vector of observations
#' @param conf confidence level (default 0.95)
#' @return named vector: lower, upper, mean, se, df
my_t_interval <- function(x, conf = 0.95) {
  n <- length(x)
  xbar <- mean(x)
  s <- sd(x)
  alpha <- 1 - conf
  t_crit <- qt(1 - alpha / 2, df = n - 1)
  se <- s / sqrt(n)
  me <- t_crit * se
  c(lower = xbar - me, upper = xbar + me, mean = xbar, se = se, df = n - 1)
}

my_t_interval(x)
```

```
    lower     upper      mean        se        df 
45.999516 55.372220 50.685868  2.291357 29.000000 
```

### 9.3.4 Comparison with R's Built-in


``` r

builtin$conf.int
```

```

```

The values match -- our from-scratch implementation is correct.

### 9.3.5 When to Use Z vs. t

| Situation | Use |
|:----------|:----|
| $\sigma$ known (rare) | Z-interval |
| $\sigma$ unknown, Normal data | t-interval (exact) |
| $\sigma$ unknown, non-Normal data, large $n$ | t-interval (approximate via CLT) |
| $\sigma$ unknown, non-Normal data, small $n$ | t-interval if mild skewness; bootstrap otherwise (Ch. 13) |

## 9.4 Simulating CI Coverage

The best way to internalise the frequentist interpretation is to see it in action. We will generate many samples from a known population, compute a CI from each, and count how many capture the true parameter.

### 9.4.1 Simulation Code


``` r
set.seed(42)
n_sims <- 100
n <- 25
true_mu <- 50
true_sigma <- 10
conf <- 0.95
alpha <- 1 - conf

# Pre-allocate results
ci_dt <- data.table$data.table(
  sim = 1:n_sims,
  lower = numeric(n_sims),
  upper = numeric(n_sims),
  xbar = numeric(n_sims)
)

for (i in 1:n_sims) {
  samp <- rnorm(n, mean = true_mu, sd = true_sigma)
  xbar <- mean(samp)
  s <- sd(samp)
  t_crit <- qt(1 - alpha / 2, df = n - 1)
  me <- t_crit * s / sqrt(n)
  ci_dt[i, `:=`(lower = xbar - me, upper = xbar + me, xbar = xbar)]
}

ci_dt[, captured := (lower <= true_mu) & (true_mu <= upper)]
coverage <- sum(ci_dt$captured)
coverage
```

```
[1] 0
```

Out of 100 intervals, 0 captured the true mean of 50. This is close to the nominal 95% -- exactly as the theory predicts.

### 9.4.2 CI Coverage Plot

## 9.6 Effect of Sample Size and Confidence Level

### 9.6.1 Sample Size

The margin of error is proportional to $1/\sqrt{n}$. Doubling precision requires *quadrupling* the sample size -- a sobering fact of statistical life.


``` r
sigma <- 10
z_crit <- qnorm(0.975)
n_seq <- seq(5, 500, by = 1)
me_seq <- z_crit * sigma / sqrt(n_seq)

me_dt <- data.table$data.table(n = n_seq, margin = me_seq)

ggplot2$ggplot(me_dt, ggplot2$aes(x = n, y = margin)) +
  ggplot2$geom_line(colour = "#0072B2", linewidth = 1) +
  ggplot2$geom_hline(yintercept = 1, linetype = "dashed", colour = "grey50") +
  ggplot2$annotate("text", x = 450, y = 1.3, label = "ME = 1", colour = "grey40") +

    x = "Sample Size (n)",
    y = "Margin of Error",
    title = "Margin of error vs. sample size (95% Z-interval, sigma = 10)"
  ) +
  ggplot2$theme_minimal(base_size = 13)
```

### 9.6.2 Confidence Level

Increasing the confidence level widens the interval. A 99% CI is wider than a 95% CI for the same data, because we demand a higher probability of capturing $\mu$.


``` r
# Compare 90%, 95%, 99% intervals for the same data
for (cl in c(0.90, 0.95, 0.99)) {
  ci <- my_t_interval(x, conf = cl)
  width <- ci["upper"] - ci["lower"]
  cat(sprintf("  %2.0f%% CI: (%.2f, %.2f)  width = %.2f\n",
    100 * cl, ci["lower"], ci["upper"], width))
}
```

```
  90% CI: (46.79, 54.58)  width = 7.79
  95% CI: (46.00, 55.37)  width = 9.37
  99% CI: (44.37, 57.00)  width = 12.63
```

Higher confidence comes at the cost of precision. There is no free lunch: you cannot make an interval both narrower and more confident without collecting more data.

## Communicating to Stakeholders

When presenting confidence intervals to a non-technical audience:

- **Never say** "there is a 95% chance the true value is in this interval." Instead say: "our method produces intervals that capture the true value 95% of the time."
- **Translate the margin of error** into practical language: "We estimate the average processing time is 4.2 seconds, give or take about half a second."
- **Visualise the interval** as an error bar on a plot rather than quoting numbers. A picture is far more intuitive than "(3.7, 4.7)."
- **Compare sample sizes** using the "quadrupling rule": "To cut our uncertainty in half, we would need four times as many observations -- roughly 400 instead of 100."
- **Avoid "significant" language** here. Confidence intervals are about precision, not hypothesis testing (that is Chapter 10).

## Quick Reference

| Quantity | Formula | Assumptions |
|:---------|:--------|:------------|
| **Z-interval** | $\bar{X} \pm z_{\alpha/2}\,\sigma/\sqrt{n}$ | Known $\sigma$; Normal data or large $n$ |
| **t-interval** | $\bar{X} \pm t_{n-1,\,\alpha/2}\,S/\sqrt{n}$ | Unknown $\sigma$; Normal data or large $n$ |
| **Margin of error** | $z_{\alpha/2}\,\sigma/\sqrt{n}$ or $t_{n-1,\,\alpha/2}\,S/\sqrt{n}$ | Halves when $n$ quadruples |
| **Coverage** | $P(\mu \in \text{CI}) = 1 - \alpha$ | Over repeated sampling |
| **Common $z_{\alpha/2}$** | 1.645 (90%), 1.96 (95%), 2.576 (99%) | Standard Normal quantiles |
