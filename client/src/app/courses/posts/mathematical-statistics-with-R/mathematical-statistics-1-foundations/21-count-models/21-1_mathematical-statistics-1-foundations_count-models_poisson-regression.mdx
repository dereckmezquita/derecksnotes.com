# Chapter 21 -- Poisson and Count Models

## Part 1: Poisson Regression

Count data are everywhere: the number of insurance claims filed per year, the number of incidents on a ship, the number of publications by a scientist. These outcomes are non-negative integers, often right-skewed, and frequently have variance that increases with the mean. Ordinary linear regression is the wrong tool -- it predicts negative counts and assumes constant variance. **Poisson regression** is the natural starting point for modelling count data.

In this chapter we derive Poisson regression from the exponential family framework developed in Chapter 19, interpret coefficients as **rate ratios**, handle exposure via offsets, and fit models to real data using `glm(..., family = poisson)`.

---

## 21.1 The Log Link as Canonical for Poisson

### 21.1.1 Review

Recall from Chapter 19 that the Poisson PMF in exponential family form is:

$$P(Y = y \mid \lambda) = \frac{1}{y!} \exp\bigl[y \log \lambda - \lambda\bigr]$$

The canonical parameter is $\theta = \log \lambda$, so the **canonical link** for the Poisson family is the **log link**:

$$g(\mu) = \log \mu$$

The inverse link is:

$$\mu = g^{-1}(\eta) = e^{\eta}$$

### 21.1.2 The Model

The Poisson regression model is:

$$Y_i \sim \text{Poisson}(\mu_i), \qquad \log \mu_i = \mathbf{x}_i^\top \boldsymbol{\beta}$$

or equivalently:

$$\mu_i = \exp(\mathbf{x}_i^\top \boldsymbol{\beta}) = \exp(\beta_0 + \beta_1 x_{i1} + \cdots + \beta_p x_{ip})$$

Because $\mu_i = E[Y_i]$ and $\text{Var}(Y_i) = \mu_i$ for the Poisson distribution, the mean-variance relationship is fixed: **the variance equals the mean**. This is both a strength (parsimony) and a weakness (rigidity) that we address in Part 2.

---

## 21.2 Deriving the Log-Likelihood

For independent observations $y_1, \ldots, y_n$:

$$\ell(\boldsymbol{\beta}) = \sum_{i=1}^n \bigl[y_i \log \mu_i - \mu_i - \log(y_i!)\bigr]$$

Substituting $\log \mu_i = \mathbf{x}_i^\top \boldsymbol{\beta}$:

$$\ell(\boldsymbol{\beta}) = \sum_{i=1}^n \bigl[y_i \mathbf{x}_i^\top \boldsymbol{\beta} - \exp(\mathbf{x}_i^\top \boldsymbol{\beta})\bigr] + \text{const}$$

The score equations are:

$$\frac{\partial \ell}{\partial \beta_j} = \sum_{i=1}^n (y_i - \mu_i) x_{ij} = 0$$

These are the same score equations we derived for the GLM in Chapter 19, confirming that Poisson regression is a special case of the GLM framework.

---

## 21.3 Interpreting Coefficients as Rate Ratios

### 21.3.1 The Multiplicative Structure

Because $\mu_i = e^{\beta_0 + \beta_1 x_{i1} + \cdots}$, a one-unit increase in $x_j$ multiplies the mean by $e^{\beta_j}$:

$$\frac{\mu(x_j + 1)}{\mu(x_j)} = e^{\beta_j}$$

This is the **rate ratio** (or **incidence rate ratio**) -- the Poisson analogue of the odds ratio in logistic regression.

- $e^{\beta_j} > 1$: predictor increases the rate.
- $e^{\beta_j} = 1$: no effect.
- $e^{\beta_j} < 1$: predictor decreases the rate.

## 21.4 Offsets: Accounting for Exposure

### 21.4.1 The Problem

Often the count depends on the amount of "exposure" -- time at risk, area surveyed, number of employees, etc. If person A was observed for 2 years and person B for 6 months, it is unfair to compare their raw counts directly. We need to model the **rate** (count per unit exposure) rather than the raw count.

### 21.4.2 The Solution: Offset

If $t_i$ is the exposure for observation $i$, we model:

$$\log \mu_i = \log t_i + \mathbf{x}_i^\top \boldsymbol{\beta}$$

The term $\log t_i$ is called an **offset** -- it enters the linear predictor with a fixed coefficient of 1. Equivalently:

$$\mu_i = t_i \cdot \exp(\mathbf{x}_i^\top \boldsymbol{\beta})$$

so $\exp(\mathbf{x}_i^\top \boldsymbol{\beta})$ is the **rate per unit exposure**.

## 21.5 Poisson Regression from Scratch

Let us implement the log-likelihood, gradient, and Newton-Raphson fitting for Poisson regression without using `glm()`.


``` r
poisson_loglik <- function(beta, X, y, offset = rep(0, length(y))) {
  eta <- X %*% beta + offset
  mu <- exp(eta)
  sum(y * eta - mu - lgamma(y + 1))
}

poisson_score <- function(beta, X, y, offset = rep(0, length(y))) {
  eta <- X %*% beta + offset
  mu <- exp(eta)
  t(X) %*% (y - mu)
}

poisson_fit <- function(X, y, offset = rep(0, length(y)), tol = 1e-8, max_iter = 50) {
  # Initialise with small positive values to avoid log(0)
  beta <- rep(0, ncol(X))
  beta[1] <- log(mean(y) + 0.1)

  for (iter in seq_len(max_iter)) {
    eta <- X %*% beta + offset
    mu <- as.vector(exp(eta))

    # Clamp mu for numerical stability
    mu <- pmax(mu, 1e-10)

    # Working weights (for Poisson with log link: w = mu)
    w <- mu

    # Working response
    z <- as.vector(eta + (y - mu) / mu)

    # IRLS update using weighted least squares via QR decomposition
    W_half <- sqrt(w)
    Xw <- W_half * X
    zw <- W_half * z
    beta_new <- as.vector(qr.solve(Xw, zw))

    if (max(abs(beta_new - beta)) < tol) {
      beta <- beta_new
      break
    }
    beta <- beta_new
  }

  eta_final <- X %*% beta + offset
  mu_final <- exp(eta_final)
  deviance <- 2 * sum(ifelse(y > 0, y * log(y / mu_final), 0) - (y - mu_final))

  list(
    coefficients = beta,
    iterations = iter,
    deviance = as.numeric(deviance),
    fitted_values = as.vector(mu_final)
  )
}

# Fit to insurance claims data
```

## 21.6 Deviance for Poisson Models

The **Poisson deviance** is:

$$D = 2 \sum_{i=1}^n \left[y_i \log\frac{y_i}{\hat{\mu}_i} - (y_i - \hat{\mu}_i)\right]$$

with the convention $0 \log 0 = 0$.


``` r
mu_hat <- fitted(fit_claims)
dev_terms <- ifelse(y_claims > 0, y_claims * log(y_claims / mu_hat), 0) - (y_claims - mu_hat)
dev_scratch <- 2 * sum(dev_terms)

c(
  "Deviance (scratch)" = round(dev_scratch, 4),
  "Deviance (glm)" = round(deviance(fit_claims), 4)
)
```


### 21.6.1 Goodness of Fit

For Poisson data with large $\mu_i$, the deviance is approximately $\chi^2$ with $n - p$ degrees of freedom under the true model. A deviance much larger than the degrees of freedom suggests **overdispersion** (see Part 2).


``` r

    deviance(fit_claims) / fit_claims$df.residual
```

## Communicating to Stakeholders

When presenting Poisson regression results:

- **Use rate ratios.** Say "customers aged 50-60 file claims at 1.3 times the rate of those aged 30-40" rather than quoting log-scale coefficients.
- **Explain offsets.** "We accounted for the fact that some policyholders were observed for longer than others by modelling the rate (claims per year) rather than the raw count."
- **Flag overdispersion.** If the deviance is much larger than the degrees of freedom, warn stakeholders that the standard errors may be too small and the model's confidence intervals too narrow.
- **Visualise predicted rates.** A curve showing how the expected count changes with a predictor is far more accessible than a table of coefficients.

## Quick Reference

| Concept | Details |
|:--------|:--------|
| **Model** | $\log \mu_i = \mathbf{x}_i^\top \boldsymbol{\beta}$ |
| **Inverse link** | $\mu_i = e^{\eta_i}$ |

| **Offset** | $\log \mu_i = \log t_i + \mathbf{x}_i^\top \boldsymbol{\beta}$ |
