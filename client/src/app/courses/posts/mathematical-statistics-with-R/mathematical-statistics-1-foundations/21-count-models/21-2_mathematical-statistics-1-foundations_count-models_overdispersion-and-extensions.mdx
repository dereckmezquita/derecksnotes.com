# Chapter 21 -- Poisson and Count Models

## Part 2: Overdispersion and Extensions

In Part 1 we fitted Poisson regression models and interpreted coefficients as rate ratios. The Poisson model imposes a strong constraint: $\text{Var}(Y) = E[Y]$. Real count data frequently violate this -- the variance exceeds the mean. This phenomenon is called **overdispersion**, and ignoring it leads to standard errors that are too small, confidence intervals that are too narrow, and $p$-values that are too small.

In this part we develop three strategies for handling overdispersion: the quasi-Poisson approach, the negative binomial model (derived as a Poisson-Gamma mixture), and zero-inflated models for data with excess zeros.

---

## 21.5 Diagnosing Overdispersion

### 21.5.1 The Dispersion Parameter

For a Poisson GLM, the **dispersion parameter** $\phi$ should equal 1. We estimate it as:

$$\hat{\phi} = \frac{1}{n - p} \sum_{i=1}^n \frac{(y_i - \hat{\mu}_i)^2}{\hat{\mu}_i} = \frac{X^2_P}{n - p}$$

where $X^2_P$ is the Pearson chi-squared statistic. If $\hat{\phi} \gg 1$, we have overdispersion. If $\hat{\phi} < 1$, we have underdispersion (rarer).

## 21.6 Quasi-Poisson

### 21.6.1 The Idea

The simplest fix is to leave the mean model unchanged but **inflate the standard errors** by $\sqrt{\hat{\phi}}$. This is the **quasi-Poisson** approach. It does not specify a full probability model; it only specifies the mean-variance relationship:

$$\text{Var}(Y_i) = \phi \cdot \mu_i$$

where $\phi$ is estimated from the data.


``` r
fit_quasi <- glm(art ~ fem + mar + kid5 + phd + ment, data = pubs, family = quasipoisson)
summary(fit_quasi)
```

### 21.6.2 Limitations

Quasi-Poisson does not define a full likelihood, so:

- AIC is not defined (R will return `NA`).
- You cannot use the LRT directly (use an $F$-test instead).
- There is no distributional assumption to check.

## 21.7 The Negative Binomial Distribution

### 21.7.1 Derivation as a Poisson-Gamma Mixture

A more principled approach to overdispersion is the **negative binomial** model. It arises naturally from a hierarchical argument:

1. Each individual has their own Poisson rate $\lambda_i$.
2. These rates vary across individuals according to a Gamma distribution: $\lambda_i \sim \text{Gamma}(\theta, \theta / \mu_i)$.
3. Marginalising over $\lambda_i$, we get:

$$Y_i \mid \mu_i, \theta \sim \text{NegBin}(\mu_i, \theta)$$

with PMF:

$$P(Y = y) = \frac{\Gamma(y + \theta)}{\Gamma(\theta)\, y!} \left(\frac{\theta}{\theta + \mu}\right)^\theta \left(\frac{\mu}{\theta + \mu}\right)^y$$

The key property is:

$$E[Y] = \mu, \qquad \text{Var}(Y) = \mu + \frac{\mu^2}{\theta}$$

The variance has an extra term $\mu^2 / \theta$ compared to Poisson. When $\theta \to \infty$, $\text{Var}(Y) \to \mu$ and we recover the Poisson. The parameter $\theta$ is sometimes called the **dispersion parameter** (confusingly, it measures the *inverse* of overdispersion: larger $\theta$ means less overdispersion).

## 21.10 Zero-Inflated Models

### 21.10.1 The Problem

Some count datasets have more zeros than any standard count model can accommodate. For example, many scientists publish zero papers not because their Poisson rate is low, but because they are in a completely different state (e.g., they changed careers). A **zero-inflated** model captures this by mixing a point mass at zero with a count distribution:

$$P(Y = 0) = \pi + (1 - \pi) \cdot P_{\text{count}}(Y = 0)$$
$$P(Y = y) = (1 - \pi) \cdot P_{\text{count}}(Y = y), \quad y = 1, 2, \ldots$$

Here $\pi$ is the probability of being a "structural zero" (someone who could never produce a positive count), and $P_{\text{count}}$ is a Poisson or negative binomial distribution for those who could.

### 21.10.2 Zero-Inflated Poisson (ZIP)


``` r
if (requireNamespace("pscl", quietly = TRUE)) {
  fit_zip <- pscl::zeroinfl(art ~ fem + mar + kid5 + phd + ment | ment + phd,
                            data = pubs, dist = "poisson")
  summary(fit_zip)
}
```

The model has two parts:
- **Count model** (above the `|`): predictors for the Poisson mean among "at-risk" scientists.
- **Zero-inflation model** (after the `|`): predictors for the probability of being a structural zero.

## 21.13 Decision Flowchart

When you encounter count data, follow this decision process:

1. **Start with Poisson regression.** It is the simplest count model.
2. **Check for overdispersion.** Compute $\hat{\phi}$. If $\hat{\phi} > 1.5$, the Poisson standard errors are unreliable.
3. **If overdispersion is mild** ($\hat{\phi} < 3$), use **quasi-Poisson** for a quick fix.
4. **If overdispersion is substantial** ($\hat{\phi} \ge 3$), fit a **negative binomial** model.
5. **If there are excess zeros**, consider a **zero-inflated** model (ZIP or ZINB).
6. **Compare models** using AIC (for NB vs Poisson) and Vuong's test (for zero-inflated vs standard).


``` r
decision_df <- data.frame(
  Scenario = c(
    "Var(Y) approx E[Y]",
    "Mild overdispersion",
    "Substantial overdispersion",
```

## Communicating to Stakeholders

When presenting count model results with overdispersion:

- **Explain overdispersion simply.** "The data are more variable than the standard Poisson model expects. This means if we use the standard model, we would overstate our confidence in the results."
- **Justify the model choice.** "We used the negative binomial model because it properly accounts for the extra variability in the data, giving us honest confidence intervals."
- **Still report rate ratios.** The coefficient interpretation is the same regardless of the model -- only the uncertainty changes.
- **For zero-inflation**: "Some scientists publish zero papers because they left academia, not because their productivity is low. The zero-inflated model separates these two groups."

## Quick Reference

| Concept | Details |
|:--------|:--------|
| **Overdispersion** | $\text{Var}(Y) > E[Y]$; $\hat{\phi} = X^2_P / (n - p) > 1$ |
| **Quasi-Poisson** | Same coefficients, inflated SE by $\sqrt{\hat{\phi}}$ |
| **Negative binomial** | $\text{Var}(Y) = \mu + \mu^2/\theta$; Poisson-Gamma mixture |
| **NB as $\theta \to \infty$** | Converges to Poisson |
| **ZIP** | $P(0) = \pi + (1-\pi) e^{-\mu}$; two-component mixture |
| **ZINB** | Zero-inflation + negative binomial count component |
| **R: quasi-Poisson** | `glm(..., family = quasipoisson)` |
| **R: negative binomial** | `MASS::glm.nb()` |
| **R: zero-inflated** | `pscl::zeroinfl(..., dist = "poisson")` or `"negbin"` |
| **Model comparison** | AIC for nested/non-nested; LRT for nested models |

## Exercises

1. **Overdispersion test.** For the biochemist publications data, perform a formal test of $H_0: \phi = 1$ vs $H_1: \phi > 1$ using the Pearson chi-squared statistic divided by the degrees of freedom. State your conclusion.

2. **NB derivation.** Starting from $Y \mid \lambda \sim \text{Poisson}(\lambda)$ and $\lambda \sim \text{Gamma}(\theta, \theta/\mu)$, derive the marginal distribution of $Y$ and show that $\text{Var}(Y) = \mu + \mu^2/\theta$.

3. **Quasi vs NB.** Fit both quasi-Poisson and negative binomial models to the epilepsy data. Compare the standard errors. Under what circumstances would you prefer one over the other?

4. **Zero-inflation detection.** For the publications data, compare the observed proportion of zeros with the expected proportion under the Poisson, NB, ZIP, and ZINB models. Which model best matches the observed zero rate?

5. **Simulation: overdispersion.** Generate 500 observations from a negative binomial distribution with $\mu = 3$ and $\theta = 1$. Fit a Poisson model and observe that $\hat{\phi} \gg 1$. Then fit a NB model and verify that $\hat{\theta} \approx 1$.

6. **Hurdle model.** A **hurdle model** treats the zero/non-zero decision separately from the positive-count distribution. Fit a hurdle Poisson model to the publications data using `pscl::hurdle()` and compare with the ZIP model using AIC.
