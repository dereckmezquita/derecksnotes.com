# Chapter 25 -- Bringing It Together

## Part 1: Workflow and Case Studies

We have covered a great deal of ground: from descriptive statistics (Chapter 1) through probability, estimation, testing, regression, GLMs, and Bayesian methods. But these tools are not meant to be used in isolation. A real statistical analysis follows a **workflow** -- a structured sequence of decisions, each informed by the previous step.

In this final chapter we codify the statistical workflow, then apply the full pipeline to two extended case studies that draw on methods from across the book.

---

## 25.1 The Statistical Workflow

Every good analysis follows seven stages:

$$\text{Question} \to \text{Design} \to \text{Collect} \to \text{Explore} \to \text{Model} \to \text{Check} \to \text{Interpret}$$

### Stage 1: Question

State the scientific question precisely. A vague question ("Is X related to Y?") produces a vague analysis. A precise question ("Does smoking during pregnancy reduce birthweight by more than 200 grams, after adjusting for maternal Age, Weight, and race?") dictates the model, the outcome, and the measure of effect.

### Stage 2: Design

Choose the study type (Chapter 24). Calculate the required sample size. Specify the primary outcome and the analysis plan **before** looking at data.

### Stage 3: Collect

Gather data according to the design. Document the process -- missing data, measurement protocols, deviations from the plan.

### Stage 4: Explore

Chapters 1-3 in action: compute summary statistics, visualise distributions, check for outliers, examine missingness patterns. **Never skip this step.** Exploration often reveals data quality issues that invalidate downstream models.

### Stage 5: Model

Select and fit a model appropriate for the data type and question. This may be a $t$-test (Chapter 11), a linear regression (Chapters 15-17), a GLM (Chapters 19-21), or a Bayesian model (Chapters 22-23).

### Stage 6: Check

Diagnose the model (Chapter 17): residual plots, influence measures, goodness-of-fit tests. If diagnostics reveal violations, return to Stage 5 and modify the model.

### Stage 7: Interpret

Translate statistical results into answers to the original question. Report effect sizes, confidence or credible intervals, and caveats. Communicate findings clearly to the intended audience.

**Key findings:**

- Each additional year of Age is associated with an increase of approximately 0.41 mmHg in systolic BP ($p < 0.001$).
- Each unit increase in BMI is associated with an increase of approximately 0.28 mmHg.

### 25.3.4 ROC Analysis


``` r
pred_probs <- predict(fit_logit, type = "response")

roc_data <- function(probs, actual, n_thresholds = 200) {
  thresholds <- seq(0, 1, length.out = n_thresholds)
  actual_bin <- as.integer(actual == "Low")
  roc_list <- lapply(thresholds, function(th) {
    pred_bin <- as.integer(probs >= th)
    tp <- sum(pred_bin == 1 & actual_bin == 1)
    fp <- sum(pred_bin == 1 & actual_bin == 0)
    fn <- sum(pred_bin == 0 & actual_bin == 1)
    tn <- sum(pred_bin == 0 & actual_bin == 0)
    data.table$data.table(
      threshold = th,
      tpr = tp / (tp + fn),
      fpr = fp / (fp + tn)
    )
  })
  data.table$rbindlist(roc_list)
}

roc_dt <- roc_data(pred_probs, birthwt$low)

auc_val <- -sum(diff(roc_dt$fpr) * (roc_dt$tpr[-1] + roc_dt$tpr[-nrow(roc_dt)]) / 2)

ggplot2$ggplot(roc_dt, ggplot2$aes(x = fpr, y = tpr)) +
  ggplot2$geom_line(colour = "#0072B2", linewidth = 1.2) +
  ggplot2$geom_abline(intercept = 0, slope = 1, linetype = "dashed", alpha = 0.5) +
  ggplot2$annotate("text", x = 0.6, y = 0.3, label = sprintf("AUC = %.3f", auc_val),
                   size = 5, colour = "#D55E00") +
  ggplot2$labs(
    x = "False Positive Rate (1 - Specificity)",
    y = "True Positive Rate (Sensitivity)",
    title = "ROC curve for low birthweight prediction"
  ) +
  ggplot2$coord_equal() +
  ggplot2$theme_minimal(base_size = 13)
```

## 25.5 Final Reflections

Statistical reasoning is not about memorising formulae. It is about asking good questions, choosing appropriate tools, checking assumptions, and communicating honestly. The best statistician is not the one who knows the most tests -- it is the one who asks: "What is the question? What does the data actually say? What are the limitations?"

We leave you with three principles:

1. **Always explore before modelling.** A plot reveals what no test statistic can.
2. **Always check your assumptions.** A model that violates its assumptions produces answers that are precisely wrong.
3. **Always report uncertainty.** A point estimate without an interval is a half-told story.

---

## Communicating to Stakeholders

- **Tell a story.** The workflow is a narrative: "We asked this question, collected this data, found this pattern, fitted this model, checked it was reasonable, and concluded X."
- **Lead with the answer.** Stakeholders want the conclusion first, then the supporting evidence.
- **Use visually intuitive plots.** Effect plots, predicted probabilities, and forest plots are more accessible than coefficient tables.
- **Acknowledge limitations.** "Our model explains 25% of the variation in blood pressure. The remaining 75% is driven by factors we did not measure." This honesty builds trust.
- **Provide actionable insights.** "Smokers have twice the odds of low birthweight babies. Smoking cessation programmes during pregnancy could meaningfully reduce this risk."

1. **Full NHANES analysis.** Extend the blood pressure analysis to include Height, Weight, and Diabetes as predictors. Perform model selection using AIC. Check diagnostics and interpret the final model.

2. **Prediction comparison.** Split the birthweight data into 70% training and 30% test. Fit the logistic regression on the training set, predict on the test set, and compute the AUC. Compare with the training-set AUC. Is there evidence of overfitting?

3. **Bayesian blood pressure.** Fit the NHANES blood pressure model using `rstanarm::stan_lm()` or `brms::brm()`. Compare posterior intervals with frequentist confidence intervals. Compute $P(\beta_{\text{Age}} > 0.5 \mid \text{data})$.

4. **Complete workflow.** Choose a dataset from the `data/` directory that we have not used in this chapter. Apply the full seven-stage workflow. Write a one-page report summarising your findings.

5. **Cross-validation.** Implement 5-fold cross-validation for the birthweight logistic regression. Report the cross-validated AUC and compare with the full-sample AUC.

6. **Book I reflection.** For each of the 25 chapters, write one sentence describing the most important concept. Then identify which three chapters are most relevant to your own research or work.
