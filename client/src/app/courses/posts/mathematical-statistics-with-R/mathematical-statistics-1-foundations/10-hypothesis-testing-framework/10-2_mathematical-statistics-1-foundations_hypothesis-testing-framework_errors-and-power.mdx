# Chapter 10 -- Hypothesis Testing Framework

## Part 2: Errors and Power

In Part 1 we developed the logic of testing and the p-value. But every decision carries risk. When we reject $H_0$, we might be wrong -- the null might actually be true. When we fail to reject, we might be missing a real effect. These two kinds of mistakes have names, precise probabilities, and a deep relationship with one another. Understanding them is essential for designing studies that can actually answer the questions we care about.

---

## 10.5 Type I and Type II Errors

### 10.5.1 The Decision Table

Every hypothesis test has four possible outcomes:

|  | **$H_0$ true** | **$H_0$ false** |
|:--|:--|:--|
| **Reject $H_0$** | Type I error (false positive) | Correct decision (true positive) |
| **Fail to reject $H_0$** | Correct decision (true negative) | Type II error (false negative) |

- **Type I error ($\alpha$):** Rejecting $H_0$ when it is actually true. The probability of this is the significance level $\alpha$.
- **Type II error ($\beta$):** Failing to reject $H_0$ when it is actually false. The probability of this depends on the true parameter value.

### 10.5.2 Formal Definitions

Let $\mathcal{R}$ be the rejection region and $\mathbf{X} = (X_1, \ldots, X_n)$ the data.

$$\alpha = P_{\theta_0}(\mathbf{X} \in \mathcal{R}) = P(\text{reject } H_0 \mid H_0 \text{ true})$$

$$\beta(\theta) = P_{\theta}(\mathbf{X} \notin \mathcal{R}) = P(\text{fail to reject } H_0 \mid \theta \text{ is the true parameter})$$

for $\theta \in \Theta_0^c$ (i.e., under the alternative).

### 10.5.3 The Trade-off

There is an unavoidable tension between $\alpha$ and $\beta$. For a fixed sample size $n$:

- Decreasing $\alpha$ (fewer false positives) *increases* $\beta$ (more false negatives). We become harder to convince, so we miss more real effects.
- Decreasing $\beta$ (fewer false negatives) *increases* $\alpha$ (more false positives). We become easier to convince, so we make more false alarms.

The only way to reduce **both** simultaneously is to increase the sample size $n$. More data gives us more information, breaking the trade-off.

### 10.5.4 Deriving $\alpha$ and $\beta$ for the Z-test

Consider $X_1, \ldots, X_n \overset{iid}{\sim} N(\mu, \sigma^2)$ with $\sigma$ known. Test $H_0: \mu = \mu_0$ vs. $H_1: \mu \neq \mu_0$ at level $\alpha$.

The rejection region is $|Z| > z_{\alpha/2}$, where $Z = (\bar{X} - \mu_0) / (\sigma/\sqrt{n})$.

**Type I error:** Under $H_0$, $Z \sim N(0, 1)$, so:

$$\alpha = P(|Z| > z_{\alpha/2}) = 2 \cdot P(Z > z_{\alpha/2}) = 2 \cdot (\alpha/2) = \alpha. \quad \checkmark$$

This is by construction -- we *chose* the critical value to give exactly this Type I error rate.

**Type II error:** Suppose the true mean is $\mu_1 \neq \mu_0$. Under $\mu_1$:

$$Z = \frac{\bar{X} - \mu_0}{\sigma/\sqrt{n}} \sim N\!\left(\frac{\mu_1 - \mu_0}{\sigma/\sqrt{n}},\; 1\right).$$

Let $\delta = (\mu_1 - \mu_0) / (\sigma/\sqrt{n})$ be the non-centrality parameter. Then:

$$\beta(\mu_1) = P(|Z| \leq z_{\alpha/2} \mid \mu = \mu_1)$$

$$= P(-z_{\alpha/2} \leq Z \leq z_{\alpha/2} \mid Z \sim N(\delta, 1))$$

$$= \Phi(z_{\alpha/2} - \delta) - \Phi(-z_{\alpha/2} - \delta).$$


``` r
# Compute beta for specific values
mu_0 <- 0
mu_1 <- 0.5
sigma <- 1
n <- 30
alpha <- 0.05

z_crit <- qnorm(1 - alpha / 2)
delta <- (mu_1 - mu_0) / (sigma / sqrt(n))

beta <- pnorm(z_crit - delta) - pnorm(-z_crit - delta)

data.frame(
  quantity = c("z_crit", "delta (non-centrality)", "beta (Type II error)", "Power (1 - beta)"),
  value = round(c(z_crit, delta, beta, 1 - beta), 4)
)
```

```
1                 z_crit 1.9600
2 delta (non-centrality) 2.7386
```

## 10.6 Power

### 10.6.1 Definition

The **power** of a test against a specific alternative $\theta_1$ is the probability of correctly rejecting $H_0$ when $\theta_1$ is the true parameter:

$$\text{Power}(\theta_1) = 1 - \beta(\theta_1) = P_{\theta_1}(\mathbf{X} \in \mathcal{R}).$$

Power is the probability of detecting a real effect. It depends on four quantities:

1. **Effect size** $|\theta_1 - \theta_0|$: larger effects are easier to detect.
2. **Sample size** $n$: more data means more power.
3. **Significance level** $\alpha$: a more lenient threshold gives higher power (but more false positives).
4. **Variability** $\sigma$: less noisy data means higher power.

### 10.6.2 The Power Function

The **power function** maps each possible parameter value to the probability of rejection:

$$\pi(\theta) = P_\theta(\mathbf{X} \in \mathcal{R}).$$

Note that $\pi(\theta_0) = \alpha$ (rejection under the null is a Type I error) and for $\theta \in \Theta_0^c$, $\pi(\theta)$ is the power. A good test has a power function that is close to $\alpha$ near $\theta_0$ and rises steeply towards 1 as $\theta$ moves away from $\theta_0$.

### 10.6.3 Deriving the Power Function for the Z-test

Continuing the two-sided z-test example:

$$\pi(\mu) = 1 - \left[\Phi(z_{\alpha/2} - \delta(\mu)) - \Phi(-z_{\alpha/2} - \delta(\mu))\right]$$

where $\delta(\mu) = (\mu - \mu_0)/(\sigma/\sqrt{n})$.


``` r
box::use(ggplot2)
box::use(data.table)

mu_0 <- 0
sigma <- 1
alpha <- 0.05
z_crit <- qnorm(1 - alpha / 2)

mu_grid <- seq(-1.5, 1.5, length.out = 300)
n_values <- c(10, 30, 100)

power_data <- data.table$rbindlist(lapply(n_values, function(n) {
  delta <- (mu_grid - mu_0) / (sigma / sqrt(n))
  power <- 1 - (pnorm(z_crit - delta) - pnorm(-z_crit - delta))
  data.table$data.table(
    mu = mu_grid,
    power = power,
    n = sprintf("n = %d", n)
  )
}))

power_data[, n := factor(n, levels = sprintf("n = %d", n_values))]

ggplot2$ggplot(power_data, ggplot2$aes(x = mu, y = power, colour = n)) +
  ggplot2$geom_line(linewidth = 1) +
  ggplot2$geom_hline(yintercept = alpha, linetype = "dashed", colour = "grey50") +
  ggplot2$annotate("text", x = -1.3, y = alpha + 0.04,
                   label = sprintf("alpha = %.2f", alpha), size = 3.5) +
  ggplot2$scale_colour_manual(values = c("#0072B2", "#D55E00", "#009E73")) +
  ggplot2$labs(
    x = expression(mu), y = "Power",
    title = "Power function for two-sided z-test",
    colour = "Sample size"
  ) +
  ggplot2$theme_minimal(base_size = 13) +
  ggplot2$theme(legend.position = "top")
```

## 10.8 Power Curves: Effect Size, Sample Size, and $\alpha$

### 10.8.1 Computing Power Curves


``` r
alpha <- 0.05
z_crit <- qnorm(1 - alpha / 2)
sigma <- 1

effect_sizes <- c(0.2, 0.5, 0.8, 1.0)  # Cohen's d values
n_grid <- seq(5, 200, by = 1)

power_grid <- data.table$rbindlist(lapply(effect_sizes, function(d) {
  delta <- d * sqrt(n_grid)  # non-centrality on the z scale
```

### 10.8.2 Sample Size Determination

A common design question is: "How many observations do I need to achieve 80% power for a given effect size?" We can derive the formula from the power equation.

For a two-sided z-test at level $\alpha$ with power $1 - \beta$ against $|\mu_1 - \mu_0| = \Delta$:

$$n = \left(\frac{(z_{\alpha/2} + z_\beta) \sigma}{\Delta}\right)^2$$


``` r
# Required sample size for 80% power
alpha <- 0.05
beta <- 0.20
sigma <- 1
delta_values <- c(0.2, 0.5, 0.8, 1.0)

z_alpha2 <- qnorm(1 - alpha / 2)
z_beta <- qnorm(1 - beta)

n_required <- ceiling(((z_alpha2 + z_beta) * sigma / delta_values)^2)

data.frame(
  effect_size = delta_values,
  n_required = n_required,
  label = c("Small", "Medium", "Large", "Very large")
)
```

```
  effect_size n_required      label
1         0.2        197      Small
2         0.5         32     Medium
3         0.8         13      Large
4         1.0          8 Very large
```

To detect a small effect ($d = 0.2$) with 80% power, we need approximately 196 observations. For a large effect ($d = 0.8$), only 13 suffice. This explains why pilot studies often fail to find small effects -- they are drastically underpowered.

## 10.10 Simulation: Power in Action




``` r
set.seed(42)
n <- 30
sigma <- 1
mu_0 <- 0
alpha <- 0.05
n_sim <- 10000

mu_values <- seq(0, 1, by = 0.1)

sim_power <- sapply(mu_values, function(mu_true) {
  p_vals <- replicate(n_sim, {
    x <- rnorm(n, mean = mu_true, sd = sigma)
    z <- (mean(x) - mu_0) / (sigma / sqrt(n))
    2 * (1 - pnorm(abs(z)))
  })
  mean(p_vals <= alpha)
})

# Theoretical power
z_crit <- qnorm(1 - alpha / 2)
theory_power <- sapply(mu_values, function(mu) {
  delta <- (mu - mu_0) / (sigma / sqrt(n))
  1 - (pnorm(z_crit - delta) - pnorm(-z_crit - delta))
})

power_comp <- data.table$data.table(
  mu = mu_values,
  simulated = sim_power,
  theoretical = theory_power
)

ggplot2$ggplot(power_comp) +
  ggplot2$geom_line(ggplot2$aes(x = mu, y = theoretical),
                    colour = "#D55E00", linewidth = 1) +
  ggplot2$geom_point(ggplot2$aes(x = mu, y = simulated),
                     colour = "#0072B2", size = 3) +
  ggplot2$labs(
    x = expression(mu[true]),
    y = "Rejection rate / Power",
    title = "Simulated (blue points) vs. theoretical (orange line) power"
  ) +
  ggplot2$theme_minimal(base_size = 13)
```

## 10.11 Interactive: Watch Power Change

<InteractivePlot
  controls={[
    { id: "d", type: "range", label: "Effect size (d)", min: 0, max: 2, step: 0.05, default: 0.5 },
    { id: "n", type: "range", label: "Sample size (n)", min: 5, max: 200, step: 1, default: 30 },
    { id: "alpha", type: "range", label: "Alpha", min: 0.001, max: 0.20, step: 0.001, default: 0.05 }
  ]}
  draw={`
    var W = canvas.width, H = canvas.height;
    ctx.clearRect(0, 0, W, H);
    var d = params.d, n = params.n, alpha = params.alpha;

    function normalCDF(x) {
      var t = 1 / (1 + 0.2316419 * Math.abs(x));
      var k = 0.3989422804014327;
      var p = k * Math.exp(-x * x / 2) *
        (t * (0.3193815 + t * (-0.3565638 + t * (1.781478 + t * (-1.821256 + t * 1.330274)))));
      return x > 0 ? 1 - p : p;
    }

    function normalPDF(x, mu, sd) {
      var z = (x - mu) / sd;
      return Math.exp(-0.5 * z * z) / (sd * Math.sqrt(2 * Math.PI));
    }

    function qnorm(p) {
      if (p <= 0) return -Infinity;
      if (p >= 1) return Infinity;
      if (p === 0.5) return 0;
      var sign = p < 0.5 ? -1 : 1;
      var pp = p < 0.5 ? p : 1 - p;
      var t = Math.sqrt(-2 * Math.log(pp));
      var c0 = 2.515517, c1 = 0.802853, c2 = 0.010328;
      var d1 = 1.432788, d2 = 0.189269, d3 = 0.001308;
      return sign * (t - (c0 + c1*t + c2*t*t) / (1 + d1*t + d2*t*t + d3*t*t*t));
    }

    var zc = qnorm(1 - alpha / 2);
    var delta = d * Math.sqrt(n);
    var power = 1 - (normalCDF(zc - delta) - normalCDF(-zc - delta));

    // Title with power readout
    ctx.fillStyle = '#0072B2'; ctx.font = 'bold 16px sans-serif'; ctx.textAlign = 'left';
    ctx.fillText('Power = ' + (power * 100).toFixed(1) + '%', 60, 20);

    var se = 1 / Math.sqrt(n);
    var mu0 = 0, mu1 = d;
    var critHi = mu0 + zc * se;
    var critLo = mu0 - zc * se;

    var xMin = Math.min(mu0 - 4*se, mu1 - 4*se);
    var xMax = Math.max(mu0 + 4*se, mu1 + 4*se);

    var pad = {l:55, r:15, t:30, b:35};
    var pw = W - pad.l - pad.r;
    var ph = H - pad.t - pad.b;

    function toX(val) { return pad.l + (val - xMin) / (xMax - xMin) * pw; }
    var yMax = Math.max(normalPDF(mu0, mu0, se), normalPDF(mu1, mu1, se)) * 1.1;
    function toY(val) { return pad.t + ph - (val / yMax) * ph; }

    // Beta region (under alt, between crits)
    ctx.fillStyle = 'rgba(0, 158, 115, 0.25)';
    ctx.beginPath();
    ctx.moveTo(toX(critLo), toY(0));
    for (var px = 0; px <= pw; px++) {
      var xv = xMin + px / pw * (xMax - xMin);
      if (xv >= critLo && xv <= critHi) {
        ctx.lineTo(toX(xv), toY(normalPDF(xv, mu1, se)));
      }
    }
    ctx.lineTo(toX(critHi), toY(0));
    ctx.closePath();
    ctx.fill();

    // Alpha region (under null, outside crits) - right tail
    ctx.fillStyle = 'rgba(213, 94, 0, 0.3)';
    ctx.beginPath();
    ctx.moveTo(toX(critHi), toY(0));
    for (var px = 0; px <= pw; px++) {
      var xv = xMin + px / pw * (xMax - xMin);
      if (xv >= critHi) ctx.lineTo(toX(xv), toY(normalPDF(xv, mu0, se)));
    }
    ctx.lineTo(toX(xMax), toY(0));
    ctx.closePath();
    ctx.fill();

    // Alpha region - left tail
    ctx.beginPath();
    ctx.moveTo(toX(xMin), toY(0));
    for (var px = 0; px <= pw; px++) {
      var xv = xMin + px / pw * (xMax - xMin);
      if (xv <= critLo) ctx.lineTo(toX(xv), toY(normalPDF(xv, mu0, se)));
    }
    ctx.lineTo(toX(critLo), toY(0));
    ctx.closePath();
    ctx.fill();

    // Null curve
    ctx.strokeStyle = '#0072B2'; ctx.lineWidth = 2;
    ctx.beginPath();
    for (var px = 0; px <= pw; px++) {
      var xv = xMin + px / pw * (xMax - xMin);

## Communicating to Stakeholders

When discussing power and errors with a non-technical audience:

- Use the legal analogy: "A Type I error is convicting an innocent person. A Type II error is acquitting a guilty person. Society deems false conviction worse, so we set a high bar for evidence -- just as we use a small alpha."
- Explain power as "the test's ability to detect a real effect." A study with 20% power has only a 1-in-5 chance of finding an effect that truly exists. This is like searching for your keys with your eyes half shut.
- When presenting sample size calculations: "We need 196 participants to have an 80% chance of detecting a small effect. With only 30 participants, we would have less than a 20% chance."
- Always present power analyses alongside study results. A non-significant result from a low-powered study tells us very little.

## Quick Reference

| Concept | Formula / Definition |
|:--------|:---------------------|
| **Type I error ($\alpha$)** | $P(\text{reject } H_0 \mid H_0 \text{ true})$ |
| **Type II error ($\beta$)** | $P(\text{fail to reject } H_0 \mid H_0 \text{ false})$ |
| **Power** | $1 - \beta = P(\text{reject } H_0 \mid H_0 \text{ false})$ |
| **Power function** | $\pi(\theta) = P_\theta(\mathbf{X} \in \mathcal{R})$ |
| **Sample size (z-test)** | $n = \left(\frac{(z_{\alpha/2} + z_\beta)\sigma}{\Delta}\right)^2$ |
| **Non-centrality** | $\delta = (\mu_1 - \mu_0)/(\sigma/\sqrt{n})$ |
| **Convention** | 80% power is standard; 90% for confirmatory studies |
