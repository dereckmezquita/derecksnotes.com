# Chapter 12 -- Multiple Testing

## Part 1: Family-Wise Error Rate and Bonferroni

In Chapter 11 we learnt to test a single hypothesis at significance level $\alpha$. When we test only one hypothesis, the probability of a false positive (Type I error) is controlled at $\alpha$ -- typically 0.05. But in modern practice we rarely test just one hypothesis. A genomics study may test 20,000 genes; a clinical trial may examine dozens of outcome variables; a psychology experiment may compare many subgroups. When we test many hypotheses simultaneously, false positives accumulate at an alarming rate.

This chapter develops the theory and practice of **multiple testing corrections** -- procedures that restore error control when we conduct many tests at once.

---

## 12.1 The Multiple Testing Problem

### 12.1.1 Why One-at-a-Time Testing Fails

Suppose we conduct $m$ independent hypothesis tests, each at significance level $\alpha$. For each test, the null hypothesis is true, so the probability of correctly *not* rejecting is $1 - \alpha$. The probability that *all* $m$ tests correctly yield non-significant results is:

$$P(\text{no false positives among } m \text{ tests}) = (1 - \alpha)^m.$$

Therefore, the probability of **at least one** false positive is:

$$\boxed{P(\text{at least one false positive}) = 1 - (1 - \alpha)^m.}$$

**Proof.** Let $A_i$ be the event that test $i$ produces a false positive ($i = 1, \ldots, m$). Since all nulls are true, $P(A_i) = \alpha$ for each $i$. By independence:

$$P\!\left(\bigcap_{i=1}^m A_i^c\right) = \prod_{i=1}^m P(A_i^c) = \prod_{i=1}^m (1 - \alpha) = (1 - \alpha)^m.$$

Taking the complement:

$$P\!\left(\bigcup_{i=1}^m A_i\right) = 1 - (1 - \alpha)^m. \quad \square$$

For $\alpha = 0.05$:

| $m$ (number of tests) | $P(\ge 1 \text{ false positive})$ |
|:-:|:-:|
| 1 | 0.050 |
| 5 | 0.226 |
| 10 | 0.401 |
| 20 | 0.642 |
| 50 | 0.923 |
| 100 | 0.994 |

With 20 tests, we have about a 64% chance of at least one false positive. With 100 tests, it is virtually certain.

### 12.1.2 Simulation: Watching False Positives Accumulate


``` r
set.seed(42)
box::use(data.table)

m_values <- c(1, 2, 5, 10, 20, 50, 100)
n_sims <- 10000
alpha <- 0.05

#' Simulate false positives under the global null
#' @param m number of independent tests
#' @param n_sims number of simulation runs
#' @param alpha significance level
#' @return proportion of runs with at least one false positive
sim_fpr <- function(m, n_sims, alpha) {
  any_fp <- vapply(seq_len(n_sims), function(i) {
    # Each test: draw p-value from Uniform(0,1) under H0
    p_vals <- runif(m)
    any(p_vals < alpha)
  }, logical(1))
  mean(any_fp)
}

results <- data.table$data.table(
  m = m_values,
  theoretical = 1 - (1 - alpha)^m_values,
  simulated = vapply(m_values, sim_fpr, numeric(1),
                     n_sims = n_sims, alpha = alpha)
)
```

## 12.2 Family-Wise Error Rate (FWER)

### 12.2.1 Definition

The **family-wise error rate** (FWER) is the probability of making at least one Type I error among a family of hypothesis tests:

$$\text{FWER} = P\!\left(\bigcup_{i \in \mathcal{I}_0} \{H_i \text{ rejected}\}\right),$$

where $\mathcal{I}_0$ is the index set of true null hypotheses. A procedure **controls FWER at level $\alpha$** if $\text{FWER} \le \alpha$.

### 12.2.2 Strong vs Weak Control

- **Weak FWER control:** $\text{FWER} \le \alpha$ when *all* nulls are true (global null).
- **Strong FWER control:** $\text{FWER} \le \alpha$ regardless of which and how many nulls are true.

Strong control is the standard requirement. All methods we present here provide strong control.

---

## 12.3 The Bonferroni Correction

### 12.3.1 The Union Bound (Boole's Inequality)

**Theorem (Union Bound).** For any events $A_1, \ldots, A_m$:

$$P\!\left(\bigcup_{i=1}^m A_i\right) \le \sum_{i=1}^m P(A_i).$$

**Proof.** We proceed by induction. For $m = 1$, the result is trivially $P(A_1) \le P(A_1)$.

For $m = 2$, the inclusion-exclusion formula gives:

$$P(A_1 \cup A_2) = P(A_1) + P(A_2) - P(A_1 \cap A_2) \le P(A_1) + P(A_2),$$

since $P(A_1 \cap A_2) \ge 0$.

Suppose the result holds for $m - 1$ events. Then:

$$P\!\left(\bigcup_{i=1}^m A_i\right) = P\!\left(\bigcup_{i=1}^{m-1} A_i \cup A_m\right) \le P\!\left(\bigcup_{i=1}^{m-1} A_i\right) + P(A_m) \le \sum_{i=1}^{m-1} P(A_i) + P(A_m) = \sum_{i=1}^m P(A_i).$$

This completes the induction. $\square$

### 12.3.2 The Bonferroni Procedure

The Bonferroni correction applies the union bound directly. Given $m$ tests with p-values $p_1, \ldots, p_m$, reject $H_i$ if and only if:

$$p_i \le \frac{\alpha}{m}.$$

**Proof of FWER control.** Let $\mathcal{I}_0$ denote the set of indices for which $H_i$ is true. Then:

$$\text{FWER} = P\!\left(\bigcup_{i \in \mathcal{I}_0} \{p_i \le \alpha/m\}\right) \le \sum_{i \in \mathcal{I}_0} P(p_i \le \alpha/m) = \sum_{i \in \mathcal{I}_0} \frac{\alpha}{m} = |\mathcal{I}_0| \cdot \frac{\alpha}{m} \le m \cdot \frac{\alpha}{m} = \alpha. \quad \square$$

Note that we used the fact that under $H_i$ true, the p-value $p_i$ is uniformly distributed on $[0, 1]$, so $P(p_i \le \alpha/m) = \alpha/m$.

### 12.3.3 Implementation from Scratch


``` r
#' Bonferroni correction -- implemented from scratch
#' @param p numeric vector of raw p-values
#' @param alpha family-wise significance level
#' @return data.table with original p-values, adjusted p-values, and decisions
my_bonferroni <- function(p, alpha = 0.05) {
  box::use(data.table)
  m <- length(p)
  adj_p <- pmin(p * m, 1)  # adjusted p-values, capped at 1
  data.table$data.table(
    test = seq_len(m),
    p_raw = p,
    p_adj = adj_p,
    reject = adj_p <= alpha
  )
}

# Test with our 20-test pure-noise example
```

## 12.4 Holm's Step-Down Procedure

### 12.4.1 Motivation

The Bonferroni correction is conservative: it uses the threshold $\alpha/m$ for *every* test, regardless of other p-values. **Holm's procedure** (1979) is uniformly more powerful -- it rejects everything Bonferroni rejects, and sometimes more -- while still controlling FWER at level $\alpha$.

### 12.4.2 The Algorithm

1. Order the p-values: $p_{(1)} \le p_{(2)} \le \cdots \le p_{(m)}$.
2. For $k = 1, 2, \ldots, m$: compare $p_{(k)}$ to $\alpha / (m - k + 1)$.
3. Let $k^*$ be the smallest index such that $p_{(k^*)} > \alpha / (m - k^* + 1)$.
4. Reject all hypotheses with indices $1, \ldots, k^* - 1$ (in the sorted order). If no such $k^*$ exists, reject all.

The thresholds progress: $\alpha/m, \; \alpha/(m-1), \; \alpha/(m-2), \; \ldots, \; \alpha/1$. Each subsequent test uses a less strict threshold -- hence "step-down."

### 12.4.3 Why Holm Dominates Bonferroni

Holm's first threshold is $\alpha/m$ -- identical to Bonferroni. If the smallest p-value passes this hurdle, Holm proceeds with the relaxed threshold $\alpha/(m-1)$. Bonferroni would still use $\alpha/m$. Therefore Holm rejects at least as many hypotheses as Bonferroni, and often more.

### 12.4.4 Implementation from Scratch


``` r
#' Holm's step-down procedure -- implemented from scratch
#' @param p numeric vector of raw p-values
#' @param alpha family-wise significance level
#' @return data.table with original p-values, adjusted p-values, and decisions
my_holm <- function(p, alpha = 0.05) {
  box::use(data.table)
  m <- length(p)
  ord <- order(p)
  p_sorted <- p[ord]

  # Adjusted p-values: cumulative max of p_(k) * (m - k + 1)
  adj_sorted <- numeric(m)
  for (k in seq_len(m)) {
    adj_sorted[k] <- p_sorted[k] * (m - k + 1)
  }
  # Enforce monotonicity: adjusted p-values must be non-decreasing
  adj_sorted <- cummax(adj_sorted)
  adj_sorted <- pmin(adj_sorted, 1)

  # Map back to original order
  adj_p <- numeric(m)
  adj_p[ord] <- adj_sorted

  data.table$data.table(
    test = seq_len(m),
    p_raw = p,
    p_adj = adj_p,
    reject = adj_p <= alpha
  )
}
```
